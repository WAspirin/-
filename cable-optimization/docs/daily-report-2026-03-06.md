## 💡 关键洞察

### 1. 禁忌搜索的核心优势
- **避免循环**：通过禁忌表记住最近的操作
- **跳出局部最优**：允许接受劣解
- **灵活性强**：可与多种邻域结构结合

### 2. 参数敏感性
- **禁忌长度**是最关键的参数
  - 太小 (≤5)：容易循环搜索
  - 太大 (≥30)：限制搜索空间，退化为贪心
  - 推荐：问题规模的 10-30%

### 3. 与 VNS 的对比
| 维度 | VNS | TS |
|------|-----|----|
| 核心机制 | 邻域切换 | 禁忌表 |
| 记忆 | 无 | 短期 + 长期 |
| 接受准则 | 只接受改进 | 可接受劣解 |
| 参数数量 | 少 (k_max) | 多 (tenure, max_iter, ...) |
| 实现难度 | 中等 | 中等偏上 |

### 4. 适用场景判断
**TS 适合**：
- 需要精细搜索的问题
- 容易陷入局部最优的问题
- 中等规模 (50-200 节点)

**VNS 适合**：
- 邻域结构清晰的问题
- 参数调优经验不足时
- 需要快速获得较好解

## 📝 学习感悟

禁忌搜索是一个非常优雅的算法，它的核心思想简单却强大：
> "记住你去过的地方，但不要因为记忆而错过更好的选择"

禁忌表模拟了人类的短期记忆，避免在原地打转；藐视准则则体现了灵活性 —— 规则是为人服务的，当有更好的选择时，可以打破规则。

与 VNS 相比，TS 的实现稍微复杂一些（需要管理禁忌表和长期记忆），但它的搜索策略更加精细。VNS 是"换个地方试试"，TS 是"我知道那里刚去过，但如果有更好的理由，我还是会去"。

在实际应用中，TS 的参数调优确实是个挑战。禁忌长度的选择需要权衡：太短容易循环，太长限制搜索。我今天的测试中，6% 的改进幅度不算高，主要是因为初始解质量已经较好。对于更复杂的问题，TS 的优势应该会更明显。
