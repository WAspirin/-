## 🔍 遇到的问题

### 问题 1: 算法对比的公平性

**现象**: Dijkstra 的成本 (59.71) 远低于其他算法 (~350-1100)

**分析**: 
- Dijkstra 解决的是**单源最短路径**问题（从起点到终点的最短路径）
- 其他算法（GA/PSO/SA 等）解决的是**TSP 类**问题（访问所有节点一次）
- 问题定义不同，导致成本不可直接对比

**解决方案**:
- 在总结中明确说明问题差异
- 对比时排除 Dijkstra，单独分析元启发式算法
- 未来实验应统一问题定义

### 问题 2: A* 表现异常差

**现象**: A* 成本 1106.60，远高于其他算法

**分析**:
- A* 实现中限制了搜索空间 (closed_set > 1000 时终止)
- 对于 TSP 类问题，A* 的状态空间爆炸
- 回退到贪心解，质量较差

**解决方案**:
- A* 更适合单源最短路径问题
- TSP 问题应使用专门的元启发式
- 在总结中说明 A* 的适用场景

### 问题 3: PSO 离散编码困难

**现象**: PSO 成本 751.61，表现不佳

**分析**:
- PSO 原本为连续优化设计
- 离散排列编码需要特殊处理（本次实现较简单）
- 速度更新操作在离散空间难以定义

**解决方案**:
- 使用专门的离散 PSO 变体（如基于交换操作）
- 或改用更适合离散问题的算法（GA/TS/VNS）

## 💡 关键洞察

### 1. 算法选择的核心原则

**问题匹配优先**:
- 最短路径问题 → Dijkstra/A*
- TSP 类回路 → VNS/TS/SA
- 连通所有节点 → MST
- 需要精确解 → MILP (小规模)

**没有"最好"的算法，只有"最合适"的算法**

### 2. 元启发式算法对比

**局部搜索类** (VNS/TS/SA):
- 优点：实现相对简单，参数少
- 缺点：可能陷入局部最优
- 适用：中等规模问题 (20-100 节点)

**群体智能类** (GA/PSO/ACO):
- 优点：全局搜索能力强，并行性好
- 缺点：参数多，调优复杂
- 适用：大规模问题 (>100 节点)

### 3. VNS 的成功因素

VNS 在本实验中表现最佳（启发式中），原因：
- **系统性邻域切换**: 避免陷入单一邻域的局部最优
- **扰动 + 局部搜索**: 平衡探索与利用
- **参数少**: k_max, max_iterations，易于调优
- **2-opt 增强**: 局部搜索使用经典的 2-opt 操作

### 4. 时间 - 质量权衡

| 算法 | 质量排名 | 时间排名 | 综合评价 |
|------|---------|---------|---------|
| VNS | #2 | #8 | ⭐⭐⭐⭐ 最佳平衡 |
| TS | #3 | #5 | ⭐⭐⭐⭐ 快速优质 |
| SA | #4 | #10 | ⭐⭐⭐ 质量好但慢 |
| ACO | #5 | #9 | ⭐⭐⭐ 中等 |
| GA | #8 | #4 | ⭐⭐ 一般 |

**推荐**:
- 追求质量：VNS 或 TS
- 追求速度：TS (0.4s)
- 平衡考虑：VNS
