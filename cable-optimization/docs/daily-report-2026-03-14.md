**对个人**:

**对他人**:

### 3. 文档与代码同等重要

**经验教训**:
- 好代码没有文档 = 难以复用
- 详细笔记帮助未来回顾
- 技术报告是项目的"名片"

**最佳实践**:
- 边写代码边写注释
- 每日记录学习心得
- 定期整理和总结

### 4. 算法选择的艺术

**核心原则**: 没有最好的算法，只有最合适的算法

**决策因素**:
1. 问题规模 (节点数)
2. 解质量要求
3. 时间限制
4. 可用信息 (图结构/动态环境)
5. 多目标需求

**推荐流程**:
```
问题规模 < 30? → MILP/Dijkstra (精确解)
    ↓
多目标需求? → NSGA-II
    ↓
动态环境? → DQN/PPO
    ↓
规模 > 100? → 分解算法
    ↓
一般场景 → VNS/TS/Memetic
```
