**关键建模创新**:
1. **道路类型分级**: 高速公路/主干道/次干道/支路 → 不同施工难度
2. **交通流量影响**: 中心区域流量高 → 施工成本增加
3. **概率性摄像头部署**: 基于交通流量决定是否需要监控
4. **负载均衡分配**: 避免汇聚节点过载

### 2. 优化算法实现 ✅

**算法 1: MST 主干网络优化**
- 使用 NetworkX 的 minimum_spanning_tree
- 边权重 = 电缆成本 × 长度 × 施工难度
- 形成低成本的主干通信网络

**算法 2: 摄像头 - 汇聚节点分配**
- 最近邻策略：地理距离优先
- 负载均衡策略：考虑容量约束
- 优先级调度：高优先级摄像头优先

**算法 3: VNS 局部优化**
- 从 MST 解开始
- 边替换邻域操作 (保持连通性)
- 典型改进：3-8%

3. **摄像头分配**: 青色点线显示摄像头到汇聚节点的连接

## 🔧 遇到的问题与解决

### 问题 1: 交通流量建模

**问题**: 如何量化交通流量对施工成本的影响？

**解决**:
```python
# 基于距离中心的远近估算流量
dist_from_center = sqrt((i - center_x)² + (j - center_y)²)
traffic_flow = 2.0 - 1.5 × (dist_from_center / max_dist)
# 流量范围：0.5 (边缘) ~ 2.0 (中心)
```

### 问题 2: 摄像头分配策略选择

**问题**: 最近邻 vs 负载均衡，哪个更好？

**解决**: 实现两种策略，默认使用负载均衡：
- 最近邻：成本低，但可能导致某些汇聚节点过载
- 负载均衡：考虑容量约束，更实用

### 问题 3: VNS 连通性保持

**问题**: 边替换时如何保证网络连通性？

**解决**:
```python
# 1. 移除一条边
# 2. 检查连通性 (nx.is_connected)
# 3. 如果不连通，找到连接两个连通分量的最佳替代边
# 4. 如果找不到，恢复原边
```
