**关键算法**:

1. **Spine-Leaf 网络架构设计**
   - 全互联拓扑：每个 Leaf 连接所有 Spine 交换机
   - 任意两点间跳数相同 (通常 2 跳)
   - 高带宽、低延迟、易扩展
   ```python
   # 每个 Leaf 连接到所有 Spine
   for leaf in leaf_racks:
       for spine in spine_racks:
           cable = Cable(source=leaf, target=spine, redundant=True)
   ```

2. **3D 空间建模**
   - 机架位置：(行，列，高度 U)
   - 电缆长度计算：水平距离 + 垂直距离 × 1.2 (走线槽余量)
   - 冷却区域：多边形边界，容量限制
   ```python
   def calculate_cable_length(source, target, include_vertical=True):
       h_dist = sqrt((x1-x2)² + (y1-y2)²)
       v_dist = abs(z1-z2) * 0.04445 if include_vertical else 0
       return (h_dist + v_dist) * 1.2  # 走线槽余量
   ```

3. **电缆类型选择策略**
   | 类型 | 带宽 | 成本 (元/米) | 延迟 (ns/m) | 适用场景 |
   |------|------|-------------|------------|----------|
   | Cat6 | 1Gbps | 2.0 | 5.0 | 服务器接入 |
   | Cat6A | 10Gbps | 3.5 | 5.0 | ToR 连接 |
   | Fiber OM3 | 40Gbps | 8.0 | 5.0 | 短距离骨干 |
   | Fiber OM4 | 100Gbps | 12.0 | 5.0 | Leaf-Spine |
   | Fiber OS2 | 400Gbps+ | 20.0 | 5.0 | 长距离骨干 |

4. **MST 路由优化 (Kruskal 算法)**
   - 构建所有可能的边
   - 按成本排序
   - 并查集实现连通性检查
   - 生成最小成本树
   ```python
   # Kruskal 算法
   edges.sort(key=lambda x: x[0])  # 按成本排序
   for cost, length, r1, r2 in edges:
       if union(r1, r2):  # 不形成环
           cables.append(Cable(...))
   ```

5. **冗余路径设计**
   - 每个服务器机架连接 2 个 Leaf (主备)
   - Spine-Leaf 全互联天然冗余
   - 冗余连接标记与成本分析
   ```python
   # 连接最近的 2 个 Leaf
   for i, (dist, leaf) in enumerate(distances[:2]):
       cable = Cable(..., redundant=(i == 1))
   ```

6. **冷却约束检查**
   ```python
   def check_cooling_constraint(rack):
       for zone in cooling_zones:
           if rack.in_zone(zone):
               return zone.available_capacity >= rack.heat_output
       return True
   ```

7. **网络性能分析**
   - 总延迟 = Σ(电缆长度 × 5ns/m)
   - 瓶颈带宽 = min(路径上所有电缆带宽)
   - 冗余度 = 冗余连接数 / 总连接数

## 💡 关键洞察

### 数据中心网络设计原则

1. **Spine-Leaf 是标准架构**
   - 全互联拓扑保证任意两点跳数相同
   - 易于水平扩展 (增加 Spine 或 Leaf)
   - 天然冗余，故障容忍度高

2. **电缆类型选择需权衡**
   - 服务器接入：Cat6A (10Gbps) 性价比最高
   - Leaf-Spine 骨干：Fiber OM4 (100Gbps) 是主流
   - 长距离/核心层：Fiber OS2 (400Gbps+)

3. **3D 布局很重要**
   - 垂直走线 (桥架) 增加长度但必需
   - 冷热通道设计影响散热效率
   - 维护通道预留空间

4. **冗余设计增加成本但值得**
   - 今日案例：62.5% 连接为冗余
   - 成本增加约 80%，但可靠性大幅提升
   - 对于关键业务数据中心是必需的

5. **冷却约束不可忽视**
   - 高功率密度机架需要精确散热计算
   - 冷却区域利用率建议控制在 75-80%
   - 预留 20-25% 余量应对峰值负载

## 🎓 学习心得

数据中心网络布线案例让我理解了**架构设计**的重要性：

> **"好的架构让优化变得简单，差的架构让优化变得不可能"**

Spine-Leaf 架构之所以成为数据中心网络的标准，不是因为它是"最优"的，而是因为它在**性能、扩展性、可靠性、成本**之间取得了最佳平衡。

这与我们学习的优化算法是相通的：
- 没有"最好"的算法，只有"最适合"问题的算法
- 架构设计决定了优化空间的上限
- 约束条件让问题更真实，也让解决方案更有价值

从海上风电场到城市电网，再到数据中心，三个案例涵盖了**能源、基础设施、IT**三大领域，让我对线缆布线优化有了更全面的理解。

感谢 WonderXi 提供的学习机会，这个项目让我从单一算法学习走向了系统化工程思维。
