### 1. 城市电网规划案例实现

**文件位置**: `examples/23_city_power_grid.py`

**案例背景**:
- 城市配电网电缆网络设计
- 多电压等级系统 (220kV/110kV/10kV/0.4kV)
- 考虑地理约束 (河流、建筑物、公园)
- N-1 可靠性分析
- 目标：最小化总成本 + 最大化可靠性

**核心功能**:

| 模块 | 功能 | 代码量 |
|------|------|--------|
| `CityPowerGrid` | 城市电网建模 | ~200 行 |
| `GridNode` | 节点类 (变电站/配电室/负荷) | ~50 行 |
| `Cable` | 电缆类 (多电压等级) | ~50 行 |
| `Obstacle` | 地理障碍区 | ~30 行 |
| 网络设计算法 | MST + VNS 优化 | ~150 行 |
| 潮流计算 | 简化前推回代法 | ~80 行 |
| N-1 可靠性分析 | 连通性测试 | ~60 行 |
| 可视化 | 拓扑图 + 成本分析 | ~100 行 |
| **总计** | - | **~720 行** |

**关键算法**:

1. **地理信息建模 (GIS)**
   - 障碍区多边形表示
   - 射线法点是否在多边形内检测
   - 路径穿越障碍检测 (采样法)
   - 距离计算 (考虑障碍惩罚因子 1.5x)

2. **分层网络设计**
   ```
   220kV 变电站 → 110kV 配电室 → 10kV 负荷点 → 0.4kV 用户
   ```

3. **MST 初始拓扑 (Kruskal)**
   - 计算所有节点对距离
   - 并查集实现连通性检查
   - 按电压等级自动选择电缆类型

4. **VNS 拓扑优化**
   - 扰动：随机边移除
   - 局部搜索：边交换
   - 迭代次数：50 次

5. **潮流计算 (简化前推回代)**
   ```python
   # 从负荷向电源累加功率
   for cable in cables:
       flow = sum(load_children)
       voltage_drop = flow * impedance / capacity
   ```

6. **N-1 可靠性分析**
   - 逐条电缆断开测试
   - BFS 连通性检查
   - 关键电缆识别
   - 冗余度评分计算

**电缆类型参数**:

| 类型 | 电压 | 容量 (MVA) | 成本 (万元/km) | 电阻 (Ω/km) |
|------|------|-----------|---------------|------------|
| HV-220 | 220kV | 200 | 150 | 0.08 |
| HV-110 | 110kV | 100 | 80 | 0.12 |
| MV-10 | 10kV | 15 | 25 | 0.35 |
| LV-04 | 0.4kV | 2 | 8 | 0.65 |

**预期结果**:

  - 电缆总长度：~45 km
  - 总成本：~1200 万元

## 💡 关键洞察

### 算法选择原则

1. **小规模精确解**: MILP (保证最优)
2. **路径规划**: Dijkstra/A* (快速精确)
3. **全局优化**: GA/PSO/SA (避免局部最优)
4. **大规模问题**: 分解 + 启发式组合
5. **动态环境**: 强化学习 (在线学习)
6. **图结构问题**: GNN (利用拓扑信息)

### 实践经验

1. **参数调优至关重要**: 同一算法不同参数性能差异可达 50%+
2. **混合策略效果好**: GA+ 局部搜索、MST+VNS 等组合优于单一算法
3. **可视化验证必要**: 图形化结果帮助发现逻辑错误
4. **实际约束让问题更真实**: 地理、电气约束使方案可落地
5. **文档与代码同等重要**: 好的文档让未来维护更高效

## 🎓 学习心得

三周的学习让我深刻理解了优化算法的本质：

> **"没有最好的算法，只有最适合问题的算法"**

- MILP 理论完美但计算昂贵
- 启发式快速但需要参数调优
- 强化学习灵活但需要大量训练
- 混合算法往往能取得最佳平衡

关键不是掌握多少算法，而是理解每种方法的**适用场景**和**局限性**。
