## 🔬 关键技术洞察

### 1. 算法选择没有银弹

**发现**: 不同场景需要不同算法

| 场景 | 推荐算法 | 理由 |
|------|----------|------|
| 小规模 (<50 节点) | Dijkstra/A* | 精确解，速度快 |
| 中等规模 (50-100) | VNS/TS | 质量好，时间可接受 |
| 大规模 (>100) | 分解算法 | 唯一可行方案 |
| 多目标 | NSGA-II | Pareto 前沿分析 |
| 动态环境 | DQN/PPO | 在线学习能力 |
| 实时应用 | Dijkstra/TS | 低延迟 |

### 2. 混合算法是趋势

**发现**: 单一算法难以兼顾探索与开发

**成功案例**:
- Memetic = GA + VNS (改进 10-15%)
- 复合 DRL = DQN + 规则启发式
- 分解 + VNS = 大规模问题求解

**设计原则**:
- 全局探索 (GA/PSO) + 局部开发 (LS)
- 学习机制 (RL) + 领域知识 (启发式)

### 3. 参数调优至关重要

**发现**: 同一算法不同参数性能差异可达 50%+

**关键参数**:
| 算法 | 关键参数 | 推荐范围 |
|------|----------|----------|
| GA | 种群大小，交叉率，变异率 | 100, 0.8, 0.1 |
| PSO | 惯性权重，学习因子 | 0.7, 2.0 |
| SA | 初始温度，冷却率 | 1000, 0.95 |
| ACO | alpha, beta, rho | 1.0, 2.0, 0.1 |

### 4. 可视化是理解的关键

**发现**: 可视化帮助理解算法行为

**有效可视化**:
- 收敛曲线 (迭代 vs 成本)
- 路径图 (解的空间分布)
- 热力图 (策略/信息素分布)
- Pareto 前沿 (多目标 trade-off)

## 🚧 遇到的问题与解决方案

### 问题 1: 依赖库版本冲突

**现象**: PyTorch/PuLP 等库版本不兼容

**解决**:
- 使用 requirements.txt 固定版本
- 虚拟环境隔离
- 优先使用 NumPy 实现核心逻辑

### 问题 2: 中文字体显示

**现象**: matplotlib 中文标签显示方框

**解决**:
```python
plt.rcParams['font.sans-serif'] = ['SimHei', 'DejaVu Sans']
plt.rcParams['axes.unicode_minus'] = False
```

### 问题 3: 大规模问题内存溢出

**现象**: 100+ 节点时 GNN/RL 模型内存不足

**解决**:
- 问题分解策略 (分治法)
- 批量处理
- 稀疏矩阵优化

### 问题 4: 算法收敛慢

**现象**: 某些算法需要数千次迭代

**解决**:
- 参数调优 (学习率、种群大小)
- 混合策略 (全局 + 局部)
- 早停机制
