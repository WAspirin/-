- ε-greedy 探索策略
- DQN 核心改进（经验回放、目标网络）

**关键公式**：
```
Q(s,a) = Q(s,a) + α * [r + γ*max_a' Q(s',a') - Q(s,a)]
```

**核心概念**：
- **Q 值**: 状态 - 动作对的期望累积奖励
- **探索 vs 利用**: ε-greedy 平衡
- **折扣因子**: γ 决定长期/短期奖励权重
- **学习率**: α 控制更新步长

## 🐛 遇到的问题

### 问题 1: PyTorch 依赖问题

**现象**: 环境缺少 PyTorch，无法运行完整 DQN

**原因**: Python 3.6.8 版本较老，pip 安装困难

**解决方案**: 
- 使用 NumPy 实现简化版 Q-Learning
- 保留核心思想（Q 值更新、ε-greedy）
- 在代码注释中说明完整 DQN 需要 PyTorch

**代码示例**：
```python
# 简化版：Q 表代替神经网络
self.q_table = np.zeros((n_states, n_actions))

# Q-Learning 更新
self.q_table[state, action] += alpha * (target - self.q_table[state, action])
```

### 问题 2: 奖励函数设计

**现象**: 初始测试中智能体经常超时或碰撞

**原因**: 奖励信号太稀疏，难以学习

**解决方案**:
- 每步惩罚：-0.1（鼓励快速到达）
- 到达奖励：+10（明确目标）
- 碰撞惩罚：-1（避免危险）
- 超时惩罚：-5（防止无限循环）

### 问题 3: 探索率衰减

**现象**: 探索率下降过快，陷入局部最优

**原因**: 衰减系数设置不当

**解决方案**:
- 初始ε = 1.0（完全探索）
- 衰减系数 = 0.995（缓慢衰减）
- 最终ε = 0.01（保留少量探索）
- 300 回合后ε ≈ 0.22（合理平衡）

## 💡 收获与洞察

### 1. 强化学习 vs 启发式算法

| 维度 | Q-Learning | VNS/TS | GA/PSO |
|------|------------|--------|--------|
| 知识需求 | 无先验 | 无先验 | 无先验 |
| 训练方式 | 在线交互 | 迭代搜索 | 种群进化 |
| 适应性 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |
| 训练时间 | ⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| 可解释性 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |

**关键区别**：
- 启发式算法：直接优化解
- 强化学习：学习策略（状态→动作映射）

### 2. Q 表的可解释性

Q 表可视化显示每个状态的最优动作方向（↑↓←→），直观展示学习到的策略。这是深度学习的"黑盒"所不具备的优势。

### 3. 奖励塑形的重要性

稀疏奖励（只有终点有奖励）导致学习困难。添加中间奖励（每步惩罚、距离奖励）可显著加速收敛。

### 4. 探索 - 利用平衡

- 高探索率：发现新策略，但效率低
- 低探索率：利用已知策略，但可能错过更优解
- 动态衰减：初期多探索，后期多利用
