## 🎯 核心洞察

### 1. 算法选择的系统性方法

通过 27 天的学习，总结出算法选择的三步骤：

```
第一步：问题规模分析
├─ < 50 节点 → 精确算法 (MILP/Dijkstra/MST)
├─ 50-200 节点 → 启发式算法 (GA/PSO/VNS)
└─ > 200 节点 → 元启发式或分解策略

第二步：约束类型分析
├─ 简单连接 → MST/Dijkstra
├─ 容量约束 → GA/VNS/TS
├─ 几何约束 → A*/GA/PSO
├─ 多目标 → NSGA-II
└─ 动态环境 → RL

第三步：资源约束分析
├─ 时间充裕 → 多算法对比实验
├─ 实时要求 → 快速算法或预训练模型
└─ 质量关键 → 混合算法 (GA+ 局部搜索)
```

### 2. 参数调优的共性规律

**调优顺序的重要性**:
- 先调影响全局的参数 (种群大小/温度)
- 再调影响搜索行为的参数 (交叉率/变异率)
- 最后调精细参数 (迭代次数/终止条件)

**经验法则的价值**:
- 种群大小：问题规模的 5-10 倍
- 交叉率：0.7-0.9 (高交叉率促进探索)
- 变异率：0.05-0.2 (低变异率保持稳定性)
- 冷却率：0.85-0.99 (慢冷却质量好但耗时)

### 3. 性能 - 质量的权衡

| 算法类别 | 解质量 | 运行时间 | 适用场景 |
|---------|-------|---------|---------|
| 精确算法 | 100% | 指数级 | 小规模问题 |
| 图算法 | 95-100% | O(n log n) | 简单连接问题 |
| 元启发式 | 85-92% | O(n²) | 中等规模通用 |
| 强化学习 | 75-85% | 训练长/推理快 | 动态环境 |
| 机器学习 | 80-85% | 训练长/推理快 | 大规模问题 |

### 4. 混合算法的优势

**GA + VNS**:
- 全局搜索 (GA) + 局部优化 (VNS)
- 比纯 GA 提升 5-15% 解质量
- 适合对解质量要求高的场景

**MST + VNS**:
- MST 提供良好初始解
- VNS 进行微调优化
- 比纯 VNS 减少 30-50% 迭代次数

### 算法推荐引擎设计

**评分公式**:
```python
score = 规模匹配 (30 分) + 约束匹配 (30 分) + 
        质量偏好 (20 分) + 鲁棒性 (10 分) - 
        实时性惩罚 (0-20 分)
```

**知识库结构**:
```python
AlgorithmProfile(
    name="算法名称",
    category="exact/graph/metaheuristic/rl/ml",
    best_for_scales=[ProblemScale.MEDIUM],
    best_for_constraints=[ConstraintType.CAPACITY],
    avg_runtime_factor=2.0,
    solution_quality=0.85,
    robustness=0.8,
    ease_of_implementation=0.6,
    recommended_params={'population_size': 100},
    notes="说明文字"
)
```

### 性能预估模型

**运行时间公式**:
```
精确算法：T = base × n^k (k=1-3, 取决于算法)
启发式算法：T = base × n² × iterations
```

**质量预估**:
```
精确算法：100%
MST: 95-98%
VNS: 88-92%
GA: 85-90%
...
```

## 💡 最佳实践总结

### 1. 实验设计原则

1. **建立基准**: 先用简单算法 (MST/Dijkstra) 建立性能基准
2. **逐步复杂**: 从简单算法开始，逐步尝试更复杂方法
3. **参数扫描**: 对关键参数进行敏感性分析
4. **多次运行**: 启发式算法至少运行 10 次取平均
5. **记录完整**: 保存所有实验配置和结果
