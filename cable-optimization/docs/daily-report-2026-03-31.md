### 1. 新案例研究：铁路网络优化

**实现文件**: `examples/33_railway_network.py` (~550 行)

**问题背景**:
- 铁路沿线信号系统电缆布线
- 车站 - 信号机 - 道岔的通信连接
- 考虑铁路安全约束 (冗余设计、防火要求)
- 优化电缆长度和铺设成本

**核心功能**:
- `RailwayNode` 数据类：车站/信号机/道岔建模
- `TrackSection` 数据类：轨道区段 (主干/支线/隧道/桥梁)
- `CableType` 枚举：4 种电缆类型 (普通信号/阻燃信号/通信/电源)
- `RailwayNetwork` 类：铁路网络建模
- `CableRoutingOptimizer` 类：基于 MST 思想的优化算法
- `RailwayVisualizer` 类：4 子图可视化

**关键特性**:
1. **节点优先级系统**: 车站 (5) > 信号机 (4) > 道岔 (3)
2. **冗余设计**: 关键节点 (车站/信号机) 自动添加备用连接
3. **场景感知电缆选择**: 
   - 隧道 → 阻燃电缆
   - 车站连接 → 通信电缆
   - 信号机连接 → 信号电缆
4. **施工难度系数**: 隧道 (2.5x) > 桥梁 (2.0x) > 支线 (1.2x) > 主干 (1.0x)

## ⚠️ 遇到的问题

### 问题 1: 隧道段电缆选型
**现象**: 初始版本未考虑隧道的特殊防火要求

**原因**: 铁路隧道内必须使用阻燃电缆 (安全规范)

**解决**: 
- 在 `select_cable_type()` 方法中优先检查轨道类型
- 隧道段强制使用 `CableType.SIGNAL_FIRE`

### 问题 2: 冗余路由生成逻辑
**现象**: 部分关键节点未生成冗余连接

**原因**: 冗余路由生成依赖于找到非主用邻居

**解决**: 
- 改进 `add_redundant_routes()` 方法
- 确保每个 `has_redundancy=True` 的节点都有备用连接

### 问题 3: 可视化布局
**现象**: 节点标签重叠，难以阅读

**解决**: 
- 使用 `annotate()` 添加偏移
- 调整字体大小为 8pt
- 考虑使用交互式可视化 (Plotly) 作为未来改进

## 🎯 关键设计决策

### 1. 节点分类与优先级
```python
# 车站：最高优先级，必须冗余
station.priority = 5, station.has_redundancy = True

# 信号机：高优先级，必须冗余
signal.priority = 4, signal.has_redundancy = True

# 道岔：中等优先级，可选冗余
switch.priority = 3, switch.has_redundancy = False
```

### 2. 电缆选型规则
```python
if track_type == 'tunnel':
    return CableType.SIGNAL_FIRE  # 隧道必须阻燃

if node_type == 'station':
    return CableType.COMMUNICATION  # 车站用通信电缆

if node_type == 'signal':
    return CableType.SIGNAL_NORMAL  # 信号机用信号电缆
```

### 3. 成本计算模型
```python
base_cost = length * cable_type.cost_per_meter
total_cost = base_cost * construction_difficulty_factor

# 难度系数
tunnel: 2.5x  # 隧道内施工困难
bridge: 2.0x  # 桥梁上施工较难
branch: 1.2x  # 支线维护成本高
```
