## 📐 一、数学规划法

### 1.1 MILP (混合整数线性规划)

**核心思想**：将问题建模为线性目标函数 + 线性约束，部分变量为整数

**标准形式**：
```
min c^T x
s.t.  Ax ≤ b
      x ≥ 0
      x_i ∈ Z (部分变量)
```

**布线问题建模**：
- **决策变量**：
  - `x_e`: 边 e 上的流量（连续）
  - `y_e`: 是否使用边 e（0-1 整数）
  
- **目标函数**：最小化总成本 `min Σ c_e * y_e`

- **约束**：
  1. 流量守恒：流入 = 流出（除源汇点）
  2. 容量限制：`x_e ≤ cap_e * y_e`
  3. 整数性：`y_e ∈ {0,1}`

**求解器**：
- Gurobi (商业，最快)
- CPLEX (商业)
- CBC (开源，PuLP 内置)
- SCIP (开源)

**优缺点**：
- ✅ 保证全局最优
- ✅ 理论成熟
- ❌ 计算复杂度高（NP-hard）
- ❌ 大规模问题求解难

### 1.2 最短路径树方法

#### Dijkstra 算法

**核心思想**：贪心策略，每次选择距离最小的未访问节点

**伪代码**：
```
dist[source] = 0, 其他为∞
PQ = 优先队列 [(0, source)]

while PQ 非空:
    (d, u) = PQ.pop()
    if u 已访问：continue
    
    for v in neighbors(u):
        new_dist = dist[u] + weight(u,v)
        if new_dist < dist[v]:
            dist[v] = new_dist
            prev[v] = u
            PQ.push((new_dist, v))
```

**时间复杂度**：O((V+E)logV)

**适用场景**：
- 单源最短路径
- 边权非负
- 需要精确解

#### A* 算法

**改进**：加入启发式函数 h(n)

**评估函数**：`f(n) = g(n) + h(n)`
- `g(n)`: 从起点到 n 的实际代价
- `h(n)`: 从 n 到终点的估计代价（如欧氏距离）

**关键**：启发式函数必须是可采纳的（不高估实际代价）

## 🔥 二、启发式算法

### 2.1 遗传算法 (GA)

**灵感**：生物进化论

**核心操作**：
1. **编码**：路径 → 染色体（排列）
2. **选择**：锦标赛/轮盘赌
3. **交叉**：OX/PMX
4. **变异**：交换/逆转

**参数调优**：
- 种群大小：50-200
- 交叉率：0.7-0.9
- 变异率：0.01-0.2
- 精英保留：5-10%

**优缺点**：
- ✅ 全局搜索能力强
- ✅ 并行性好
- ❌ 收敛速度慢
- ❌ 参数敏感

### 2.2 粒子群优化 (PSO)

**灵感**：鸟群觅食

**速度更新**：
```
v = w*v + c1*r1*(pbest-x) + c2*r2*(gbest-x)
x = x + v
```

- `w`: 惯性权重
- `c1, c2`: 学习因子
- `pbest`: 个体最优
- `gbest`: 全局最优

### 2.3 模拟退火 (SA)

**灵感**：金属退火

**Metropolis 准则**：
```
如果 ΔE < 0: 接受新解
否则：以概率 exp(-ΔE/T) 接受
```

**退火计划**：
- 初始温度：高
- 降温系数：0.9-0.99
- 终止温度：接近 0

### 2.4 变邻域搜索 (VNS)

**核心思想**：系统性地改变邻域结构

**流程**：
```
for k = 1 to k_max:
    扰动：在当前解的第 k 个邻域随机扰动
    局部搜索：在新邻域内搜索
    如果改进：接受并重置 k=1
    否则：k = k+1
```

### 2.5 禁忌搜索 (Tabu Search, TS) ⭐ NEW

**核心思想**：使用禁忌表记录最近访问过的解或操作，避免循环；允许接受劣解以跳出局部最优

**灵感**：人类记忆机制 —— 记住最近去过的地方，避免重复访问

**核心机制**：

1. **禁忌表 (Tabu List)**
   - 记录最近执行的操作或访问的解
   - 固定长度（禁忌长度 tabu_tenure）
   - 先进先出 (FIFO) 更新

2. **藐视准则 (Aspiration Criterion)**
   - 如果新解优于历史最优，即使在禁忌表中也可接受
   - 避免错过优质解

3. **长期记忆 (Frequency Memory)**
   - 记录解或操作的访问频率
   - 用于多样化策略（选择低频操作）

**算法流程**：
```
初始化：生成初始解，清空禁忌表
while 未满足终止条件:
    生成邻域解集合
    选择最佳非禁忌解（或满足藐视准则）
    更新当前解
    将对应操作加入禁忌表
    如果优于历史最优：更新历史最优
    如果长期无改进：执行多样化策略
```

**邻域操作**：
- **Swap**: 交换两个位置
- **Insert**: 插入操作（将一个元素移到另一位置）
- **Reverse**: 逆转子序列
- **2-opt**: 经典 TSP 邻域操作

**参数调优**：
- **禁忌长度**: 问题规模的 10-30% (如 n=50, tenure=5-15)
  - 太小：容易循环
  - 太大：限制搜索空间
- **邻域大小**: 20-50 个采样
- **最大迭代**: 500-2000
- **无改进限制**: 100-300 次

**优缺点**：
- ✅ 避免循环搜索
- ✅ 允许接受劣解，跳出局部最优
- ✅ 灵活性强，可与多种邻域结合
- ❌ 参数敏感（禁忌长度）
- ❌ 需要设计合适的邻域结构

**与 VNS 对比**：
| 特性 | VNS | TS |
|------|-----|----|
| 邻域切换 | 系统性 (k=1→k_max) | 固定邻域集合 |
| 记忆机制 | 无 | 禁忌表 + 长期记忆 |
| 接受准则 | 只接受改进 | 可接受劣解 |
| 避免循环 | 邻域变化 | 禁忌表 |

**适用场景**：
- 组合优化问题（TSP、调度、布线）
- 中等规模问题 (50-200 节点)
- 需要精细搜索的问题

**线缆布线应用**：
```python
# 编码：节点访问顺序的排列
solution = [0, 5, 3, 8, 1, 7, ...]

# 邻域：交换两个节点位置
neighbor = swap(solution, i=2, j=5)

# 禁忌：记录最近交换的位置对
tabu_list.append(('swap', 2, 5))

# 藐视：如果新解成本 < 历史最优，接受
if new_cost < best_cost:
    accept(neighbor)  # 即使在禁忌表中
```

**实现提示**：
1. 使用 deque 实现禁忌表（自动 FIFO）
2. 用 tuple 作为禁忌表键（操作类型 + 参数）
3. 长期记忆用 dict 记录频率
4. 多样化策略：选择低频操作

### 2.2 粒子群优化 (PSO) - 详细实现

**算法流程**:
```
1. 初始化粒子群（随机位置和速度）
2. 评估每个粒子的适应度
3. 更新个体最优 (pbest) 和全局最优 (gbest)
4. 更新速度：v = w*v + c1*r1*(pbest-x) + c2*r2*(gbest-x)
5. 更新位置：x = x + v
6. 重复 2-5 直到收敛
```

**关键参数**:
- `w` (惯性权重): 0.4-0.9，平衡全局/局部搜索
- `c1` (个体学习因子): 通常 1.5-2.0
- `c2` (群体学习因子): 通常 1.5-2.0
- 种群大小：30-50

**线缆布线应用**:
- 优化路径控制点坐标
- 连续变量优化
- 可加入障碍物惩罚项

**代码示例**: `examples/04_pso.py`

### 1.3 A* 搜索算法 - 详细实现

**评估函数**: `f(n) = g(n) + h(n)`
- `g(n)`: 从起点到 n 的实际代价
- `h(n)`: 从 n 到终点的启发式估计

**常用启发式**:
| 启发式 | 公式 | 适用场景 |
|--------|------|----------|
| 欧氏距离 | √(dx²+dy²) | 任意方向移动 |
| 曼哈顿距离 | |dx|+|dy| | 四方向移动 |
| 切比雪夫距离 | max(|dx|,|dy|) | 八方向移动 |
| Octile 距离 | max+ (√2-1)*min | 八方向精确估计 |

**可采纳性**: 启发式必须不高估实际代价，才能保证找到最优路径

**搜索效率**:
- 时间复杂度：O(b^d)，b 为分支因子，d 为深度
- 空间复杂度：O(b^d)，需要存储 OpenSet

**线缆布线应用**:
- 网格环境路径规划
- 避开障碍物
- 机房/工厂布局布线

**代码示例**: `examples/06_astar.py`

## 📅 Day 3 新增内容 (2026-03-04)

### 1.4 最小生成树 (Minimum Spanning Tree, MST)

**问题定义**:
在连通加权无向图 G=(V,E) 中，找到一棵包含所有顶点的树 T，使得树上所有边的权重之和最小。

**性质**:
- 包含 V 个顶点和 V-1 条边
- 连通且无环
- 总权重最小

---

#### Prim 算法

**核心思想**: 贪心策略，从单个顶点开始，逐步扩展树

**算法流程**:
```
1. 选择任意起始顶点 s，加入集合 S
2. 维护优先队列 PQ，存储所有连接 S 和 V-S 的边
3. while |S| < |V|:
     从 PQ 中取出权重最小的边 (u, v)，其中 u∈S, v∉S
     将 v 加入 S，边 (u, v) 加入 MST
     将 v 的所有邻边加入 PQ
4. 返回 MST
```

**时间复杂度**:
- 使用二叉堆：O(E log V)
- 使用斐波那契堆：O(E + V log V)

**空间复杂度**: O(V + E)

**适用场景**:
- 稠密图（边多）
- 需要逐步构建 MST
- 适合邻接表/邻接矩阵表示

---

#### Kruskal 算法

**核心思想**: 贪心策略，按边权重排序，依次选择不形成环的边

**算法流程**:
```
1. 将所有边按权重从小到大排序
2. 初始化并查集，每个顶点为独立集合
3. for 每条边 (u, v) in 排序后的边:
     if find(u) != find(v):  # 不在同一集合
       将边 (u, v) 加入 MST
       union(u, v)  # 合并集合
4. 返回 MST
```

**并查集优化**:
- **路径压缩**: find 操作时压缩路径
- **按秩合并**: 小树合并到大树

**时间复杂度**: O(E log E) 或 O(E log V)
- 排序：O(E log E)
- 并查集操作：O(E α(V))，α为反阿克曼函数（几乎为常数）

**空间复杂度**: O(V + E)

**适用场景**:
- 稀疏图（边少）
- 边已排序或易于排序
- 实现简单

---

#### 两种算法对比

| 特性 | Prim | Kruskal |
|------|------|---------|
| 贪心策略 | 顶点扩展 | 边选择 |
| 数据结构 | 优先队列 | 并查集 |
| 时间复杂度 | O(E log V) | O(E log E) |
| 适合图类型 | 稠密图 | 稀疏图 |
| 实现难度 | 中等 | 简单 |
| 是否需要连通 | 是 | 是（可检测） |

#### 线缆布线应用

**场景**: 园区/建筑网络布线

**问题建模**:
- **顶点**: 建筑/设备位置
- **边**: 可能的布线路径
- **权重**: 布线成本（距离 × 单位成本）

**优势**:
- 保证所有节点连通
- 总成本最小
- 无冗余连接（无环）

**实际考虑**:
1. **冗余需求**: MST 无冗余，实际可能需要备份线路
2. **容量限制**: 某些边可能有容量上限
3. **地形因素**: 权重可包含地形难度系数
4. **已有设施**: 部分边可能已存在（权重为 0）

**扩展**:
- **Steiner Tree**: 允许添加中间节点（NP-hard）
- **度约束 MST**: 限制节点最大度数
- **多目标 MST**: 同时优化成本、延迟等

```python
            parent[x] = self.find(parent[x])  # 路径压缩
        return parent[x]

        px, py = self.find(x), self.find(y)
        if px == py:
            return False
        # 按秩合并
```

### 2.5 变邻域搜索 (Variable Neighborhood Search, VNS) - 详细实现

**核心思想**:
- 系统性地改变邻域结构，避免陷入局部最优
- 通过不同邻域的切换，实现全局搜索与局部搜索的平衡
- "如果当前邻域找不到更优解，就尝试更大的邻域"

**算法流程**:
```
1. 初始化当前解 x
2. 设置 k = 1 (从最小邻域开始)
3. while 未满足终止条件:
     a. 扰动 (Shaking): 在 x 的第 k 个邻域内随机生成 x'
     b. 局部搜索：在 x' 的邻域内搜索最优解 x''
     c. 判断：
        - 如果 f(x'') < f(x): 接受 x = x'', 重置 k = 1
        - 否则：k = k + 1 (尝试更大的邻域)
        - 如果 k > k_max: k = 1 (循环)
4. 返回最优解
```

**关键特点**:
- **扰动 (Shaking)**: 随机性帮助跳出局部最优
- **局部搜索**: 确定性搜索保证邻域内最优
- **邻域切换**: 系统性地探索不同邻域结构

---

#### 邻域结构设计 (TSP/路径问题)

**N1: 交换相邻 (Swap Adjacent)**
```
交换路径中相邻的两个节点
复杂度：O(n) 个邻居
```
示例：`[0,1,2,3,4]` → 交换 idx=2 → `[0,1,3,2,4]`

**N2: 交换任意 (Swap Any)**
```
交换路径中任意两个节点
复杂度：O(n²) 个邻居
```
示例：`[0,1,2,3,4]` → 交换 idx=1,3 → `[0,3,2,1,4]`

**N3: 路径逆转 (Reverse Segment)**
```
逆转路径中的一段
复杂度：O(n²) 个邻居
```
示例：`[0,1,2,3,4,5]` → 逆转 2-4 → `[0,1,4,3,2,5]`

**N4: 插入操作 (Insert)**
```
将一个节点移动到另一位置
复杂度：O(n²) 个邻居
```
示例：`[0,1,2,3,4]` → 插入 3→1 → `[0,3,1,2,4]`

**N5: 2-opt**
```
经典 TSP 邻域，逆转两段之间的路径
复杂度：O(n²) 个邻居
```
示例：`[0,1,2,3,4,5]` → 2-opt(2,4) → `[0,1,4,3,2,5]`

---

#### 参数调优

| 参数 | 推荐范围 | 说明 |
|------|----------|------|
| k_max | 3-7 | 邻域数量，太少易陷局部最优，太多计算慢 |
| max_iterations | 500-2000 | 最大迭代次数 |
| max_no_improve | 50-200 | 无改进最大迭代次数（提前终止） |
| 局部搜索深度 | 全搜索/部分搜索 | 全搜索质量好但慢 |

**调优策略**:
1. **从小邻域开始**: k=1 计算快，先尝试小改进
2. **渐进扩大**: 找不到改进时扩大邻域
3. **及时重置**: 找到改进后重置 k=1，充分利用小邻域

---

#### 优缺点分析

**优点**:
- ✅ 简单易实现，参数少
- ✅ 系统性地避免局部最优
- ✅ 适用于多种优化问题（TSP、VRP、调度等）
- ✅ 理论基础强（Mladenović & Hansen, 1997）
- ✅ 可扩展为 General VNS、Variable Neighborhood Descent

**缺点**:
- ❌ 邻域设计需要领域知识
- ❌ 局部搜索可能耗时（尤其是大邻域）
- ❌ 收敛速度可能慢于某些元启发式
- ❌ 参数 k_max 选择影响性能

---

#### 线缆布线应用

**场景**: 多设备顺序布线（类似 TSP）

**问题建模**:
- **解表示**: 设备访问顺序的排列
- **目标函数**: 总布线长度（或成本）
- **约束**: 所有设备必须访问一次

**优势**:
- 适合中等规模问题（20-100 节点）
- 能找到高质量近似解
- 可与其它算法混合（如 VNS+GA）

**实际应用建议**:
1. **初始解**: 使用最近邻启发式生成较好初始解
2. **邻域选择**: 从简单邻域开始（N1, N2），逐步增加
3. **混合策略**: VNS 后可接 2-opt 或 3-opt 进一步优化
4. **并行化**: 多个初始解并行运行 VNS

---

#### 代码实现要点

**VNS 主循环**:
```python
k = 1
while iteration < max_iter:
    # 1. 扰动
    new_sol = shake(current_sol, k)
    
    # 2. 局部搜索
    new_sol = local_search(new_sol, k)
    
    # 3. 判断是否接受
    if cost(new_sol) < best_cost:
        best_sol = new_sol
        k = 1  # 重置
    else:
        k += 1
        if k > k_max:
            k = 1
```

**局部搜索（最陡下降）**:
```python
def local_search(solution, k):
    best = solution
    best_cost = cost(solution)
    
    for neighbor in neighborhood(solution, k):
        if cost(neighbor) < best_cost:
            best = neighbor
            best_cost = cost(neighbor)
    
    return best
```

### 2.6 蚁群优化算法 (Ant Colony Optimization, ACO) - 详细实现

**核心思想**:
- 模拟蚂蚁觅食行为：蚂蚁在路径上释放信息素，后续蚂蚁倾向于选择信息素浓度高的路径
- **正反馈机制**：好路径 → 更多蚂蚁选择 → 更多信息素 → 更多蚂蚁
- **启发式引导**：蚂蚁也考虑距离等启发式因素（短路径优先）
- **信息素蒸发**：避免过早收敛到局部最优，保持探索能力

**灵感来源**：
> 蚂蚁个体能力有限，但群体通过信息素通信能找到从巢穴到食物的最短路径

**算法流程**:
```
1. 初始化信息素矩阵（所有边相同浓度）
2. while 未满足终止条件:
     a. 每只蚂蚁构建完整路径（概率选择下一节点）
     b. 计算每只蚂蚁的路径成本
     c. 更新信息素：
        - 蒸发：所有信息素 × (1 - rho)
        - 沉积：每只蚂蚁在其路径上留下 Q / cost
        - (可选) 精英蚂蚁额外强化最优路径
3. 返回历史最优解
```

---

#### 概率选择模型

**选择概率公式**:
```
P(i,j) = [τ(i,j)^α × η(i,j)^β] / Σ[k∈allowed] [τ(i,k)^α × η(i,k)^β]

其中:
- τ(i,j): 边 (i,j) 上的信息素浓度
- η(i,j): 启发式值，通常为 1/d(i,j)（距离的倒数）
- α: 信息素重要程度因子（α 越大越依赖信息素）
- β: 启发式因子重要程度（β 越大越贪心）
- allowed: 未访问节点集合
```

**参数影响**:
| 参数 | 值太小 | 值太大 | 推荐范围 |
|------|--------|--------|----------|
| α (信息素因子) | 退化为贪心算法 | 过早收敛到局部最优 | 0.5-2.0 |
| β (启发式因子) | 随机搜索 | 纯贪心，失去多样性 | 1.0-5.0 |
| ρ (蒸发率) | 收敛慢，信息素积累过多 | 失去记忆，退化为随机 | 0.1-0.5 |

---

#### 信息素更新策略

**1. 基本更新规则 (Ant System)**:
```
τ(i,j) = (1 - ρ) × τ(i,j) + Σ[k=1 to m] Δτ_k(i,j)

其中:
- ρ: 蒸发率 (0 < ρ ≤ 1)
- m: 蚂蚁数量
- Δτ_k(i,j) = Q / L_k  如果蚂蚁 k 经过边 (i,j)
             0        否则
- Q: 信息素强度系数
- L_k: 蚂蚁 k 的路径总长度
```

**2. 精英蚂蚁策略 (Elitist Strategy)**:
```
额外强化历史最优路径:
Δτ_best(i,j) = e × Q / L_best  如果 (i,j) 在最优路径上

其中 e 是精英蚂蚁数量（通常 5-10）
```

**3. 最大 - 最小蚂蚁系统 (MMAS)**:
```
限制信息素范围 [τ_min, τ_max]:
- 避免某些边信息素过高（垄断）
- 避免某些边信息素过低（被忽略）
- 定期重置信息素（停滞时）
```

---

#### 关键机制详解

**1. 正反馈 (Positive Feedback)**:
- 短路径 → 成本低 → 沉积信息素多 → 吸引更多蚂蚁
- 形成"富者愈富"效应，加速收敛

**2. 信息素蒸发 (Evaporation)**:
- 模拟自然界信息素挥发
- 作用：
  - 避免过早收敛（忘记坏路径）
  - 鼓励探索新路径
  - 适应动态变化的环境

**3. 启发式信息 (Heuristic Information)**:
- 提供问题特定的先验知识
- 对于 TSP/布线：η(i,j) = 1/d(i,j)
- 平衡"利用已知好路径"与"探索新路径"

**4. 精英策略 (Elitism)**:
- 额外强化历史最优解
- 加速收敛，但可能增加早熟风险
- 推荐精英蚂蚁数：5-10

---

#### 参数调优指南

**推荐初始配置**:
```python
config = ACOConfig(

    alpha=1.0,          # 信息素重要性
    beta=2.0,           # 启发式重要性（通常 > alpha）
    rho=0.1,            # 蒸发率（10% 每代）
    Q=1.0,              # 信息素强度

```

1. **先调 β**: 增大 β 使搜索更贪心，减小使搜索更随机

3. **最后调 ρ**: 收敛太快→增大 ρ，收敛太慢→减小 ρ
4. **蚂蚁数量**: 通常设为节点数的 1-2 倍

**经验法则**:
- α=1, β=2-5 是常用组合
- ρ=0.1 适合大多数问题
- 精英蚂蚁数 = 总蚂蚁数的 10-20%

#### 线缆布线应用

**问题建模**:
- **节点**: 设备/接线盒位置
- **边**: 可能的布线路径
- **目标**: 访问所有设备一次，最小化总布线长度（TSP 变体）

**编码方式**:
```python
# 路径表示：节点访问顺序
solution = [0, 5, 3, 8, 1, 7, ...]  # 排列编码

# 信息素矩阵：n × n
pheromone[i, j] = 边 (i→j) 的信息素浓度

# 启发式矩阵：n × n
heuristic[i, j] = 1 / distance(i, j)
```

**实际应用技巧**:
1. **最近邻初始化**: 用贪心算法生成部分初始解，加速收敛
2. **局部搜索增强**: ACO 生成解后，用 2-opt 进一步优化
3. **自适应参数**: 根据收敛状态动态调整 α, β, ρ
4. **多起点策略**: 蚂蚁从不同起点出发，增加多样性

#### 代码实现要点

**ACO 主循环**:
```python
for iteration in range(n_iterations):
    # 1. 每只蚂蚁构建路径
    solutions = []
    for ant in range(n_ants):
        path = construct_solution(pheromone, heuristic, alpha, beta)
        solutions.append(path)
    
    # 2. 计算成本
    costs = [compute_cost(path, dist_matrix) for path in solutions]
    
    # 3. 更新历史最优
    best_idx = argmin(costs)
    if costs[best_idx] < best_cost:
        best_cost = costs[best_idx]
        best_solution = solutions[best_idx]
    
    # 4. 更新信息素
    pheromone *= (1 - rho)  # 蒸发
    for path, cost in zip(solutions, costs):
        delta = Q / cost
        for i in range(len(path)-1):
            pheromone[path[i], path[i+1]] += delta  # 沉积
    
    # 5. 精英蚂蚁额外强化
    if elite_ants > 0:
        elite_delta = elite_ants * Q / best_cost
        for i in range(len(best_solution)-1):
            pheromone[best_solution[i], best_solution[i+1]] += elite_delta
```

**路径构造（轮盘赌选择）**:
```python
def select_next(current, visited, pheromone, heuristic, alpha, beta):
    available = [i for i in range(n) if i not in visited]
    
    # 计算选择概率
    probs = []
    for node in available:
        tau = pheromone[current, node] ** alpha
        eta = heuristic[current, node] ** beta
        probs.append(tau * eta)
    
    # 归一化 + 轮盘赌
    probs = array(probs) / sum(probs)
    return choice(available, p=probs)
```

### 3.2 深度强化学习 (DQN/Q-Learning) - 详细实现

**核心思想**：通过与环境交互学习最优策略，无需先验知识

**强化学习基本框架**：
```
智能体 (Agent) ←→ 环境 (Environment)
    ↓                    ↓
  动作 (Action)       状态 (State)
    ↓                    ↓
                    奖励 (Reward)
```

**Q-Learning 算法**：

1. **Q 值定义**：
   - Q(s, a) = 在状态 s 执行动作 a 的期望累积奖励
   
2. **Bellman 方程**：
   ```
   Q(s,a) = Q(s,a) + α * [r + γ*max_a' Q(s',a') - Q(s,a)]
   ```
   - α (alpha): 学习率 (0.1-0.3)
   - γ (gamma): 折扣因子 (0.9-0.99)
   - r: 即时奖励
   - s': 下一状态

3. **ε-greedy 策略**：
   ```
   以概率 ε：随机探索 (exploration)
   以概率 1-ε：选择最优动作 (exploitation)
   ```
   - ε 随训练衰减 (1.0 → 0.01)

**DQN 改进**（完整版本）：
- 使用神经网络近似 Q 值函数 Q(s,a;θ)
- 经验回放 (Experience Replay)：打破样本相关性
- 目标网络 (Target Network)：稳定训练
- Dueling 架构：分离状态价值和动作优势

**线缆布线问题建模**：

| 组件 | 定义 | 说明 |
|------|------|------|
| 状态 (State) | 智能体位置 + 目标位置 + 障碍物地图 | 可离散化或连续表示 |
| 动作 (Action) | 上/下/左/右移动 | 离散动作空间 |
| 奖励 (Reward) | 到达终点 +10, 碰撞 -1, 每步 -0.1 | 鼓励快速到达 |
| 终止条件 | 到达终点/碰撞/超时 | 结束回合 |

**参数调优**：
- **学习率 α**: 0.1-0.3（太大不稳定，太小收敛慢）
- **折扣因子 γ**: 0.9-0.99（重视长期奖励）
- **探索率 ε**: 初始 1.0，衰减到 0.01
- **衰减系数**: 0.99-0.999 每回合

**优缺点**：
- ✅ 无需先验知识，从零学习
- ✅ 适应动态环境
- ✅ 可处理复杂状态空间（DQN）
- ❌ 训练时间长
- ❌ 样本效率低
- ❌ 超参数敏感

**与启发式算法对比**：
| 特性 | Q-Learning | GA/PSO | VNS/TS |
|------|------------|--------|--------|
| 学习方式 | 在线交互 | 种群进化 | 单解搜索 |
| 先验知识 | 不需要 | 不需要 | 不需要 |
| 训练时间 | 长 | 中等 | 短 |
| 适应性 | 强 | 中 | 弱 |
| 可解释性 | 中 | 中 | 高 |

**代码示例**: `examples/12_dqn_reinforcement_learning.py`

**实现提示**：
1. 状态离散化：连续空间 → 分箱 (binning)
2. Q 表初始化：零初始化或乐观初始化
3. 奖励塑形：稀疏奖励 → 稠密奖励（如距离奖励）
4. 经验回放：存储 (s,a,r,s',done) 元组

**测试结果**（10x10 网格，15 个障碍物）：

- 50-150 回合：学习基本策略，奖励提升

**改进方向**：

## 🧠 五、策略梯度方法 - PPO

### 5.1 PPO (Proximal Policy Optimization)

**论文**: Schulman et al. (2017). Proximal Policy Optimization Algorithms

**核心思想**: PPO 是一种 Actor-Critic 算法，通过限制策略更新幅度来保证训练稳定性

**灵感**: 信任区域优化 (Trust Region Optimization) —— 每次更新不要走太远

#### 核心机制

**1. Actor-Critic 架构**:
```
┌─────────────┐     ┌─────────────┐
│   Actor     │     │   Critic    │
│  (策略网络)  │     │  (价值网络)  │
│             │     │             │
│  输入：状态  │     │  输入：状态  │
│  输出：动作  │     │  输出：价值  │
│  概率分布   │     │  估计 V(s)  │
└─────────────┘     └─────────────┘
```

- **Actor**: 学习策略 π(a|s)，决定采取什么动作
- **Critic**: 学习价值函数 V(s)，评估当前状态好坏

**2. 优势函数 (Advantage Function)**:
```
A(s,a) = Q(s,a) - V(s)
```
- 表示动作 a 相对于平均水平的优势
- 使用 GAE (Generalized Advantage Estimation) 估计

**3. GAE 优势估计**:
```
δ_t = r_t + γ*V(s_{t+1}) - V(s_t)  # TD 误差
A_t = δ_t + (γ*λ)*δ_{t+1} + (γ*λ)²*δ_{t+2} + ...
```
- λ (lambda): GAE 参数 (0.9-0.95)
- 平衡偏差和方差

**4. PPO 截断 (Clipping)**:
```
L^CLIP(θ) = E[min(r_t(θ)*A_t, clip(r_t(θ), 1-ε, 1+ε)*A_t)]
```
- r_t(θ) = π_θ(a|s) / π_θ_old(a|s)  (概率比)
- ε (epsilon): 截断参数 (0.1-0.2)
- 防止策略更新过大导致崩溃

#### 算法流程

```
初始化 Actor 和 Critic 网络

for each episode:
    收集轨迹 {(s_t, a_t, r_t)}
    
    计算优势函数 A_t (使用 GAE)
    计算 TD 目标 (用于 Critic)
    
    for epoch in 1..n_epochs:
        # 更新 Actor (策略梯度)
        计算概率比 r_t(θ)
        计算截断损失 L^CLIP
        梯度上升更新 Actor
        
        # 更新 Critic (价值回归)
        计算价值损失 L = (V(s) - V_target)²
        梯度下降更新 Critic
    
    如果策略变化过大：提前停止
```

#### 参数调优

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 学习率 (Actor) | 0.0003-0.001 | 太小收敛慢，太大不稳定 |
| 学习率 (Critic) | 0.001-0.003 | Critic 可以比 Actor 快 |
| 折扣因子 γ | 0.99 | 重视长期奖励 |
| GAE 参数 λ | 0.95 | 平衡偏差和方差 |
| 截断参数 ε | 0.2 | 限制更新幅度 |
| 更新轮数 n_epochs | 10 | 每批数据更新次数 |
| 隐藏层维度 | 64-256 | 网络容量 |

#### 与 DQN 对比

| 特性 | DQN | PPO |
|------|-----|-----|
| 类型 | Value-based | Policy-based |
| 输出 | Q 值 | 动作概率 |
| 动作空间 | 离散 | 离散/连续 |
| 探索 | ε-greedy | 随机策略 |
| 样本效率 | 低 | 中等 |
| 稳定性 | 中等 | 高 (截断机制) |
| 训练速度 | 快 | 中等 |

#### 优缺点

**优点**:
- ✅ 训练稳定（截断机制防止崩溃）
- ✅ 样本效率较高（多轮更新）
- ✅ 适用于连续动作空间
- ✅ 超参数相对不敏感

**缺点**:
- ❌ 实现复杂度高于 DQN
- ❌ 需要调优的参数较多
- ❌ 训练时间仍然较长

#### 在线缆布线中的应用

**状态表示**:
- 智能体当前位置 (row, col)
- 目标位置
- 局部障碍物地图

**动作空间**:
- 离散：上/下/左/右 (4 个动作)
- 连续：移动方向和距离

**奖励设计**:
```

碰撞障碍物：-1
每步惩罚：-0.1

```

   - 前 100 回合：负奖励（随机探索）

#### 图卷积网络 (Graph Convolutional Network, GCN)

**论文**: Kipf & Welling (2017). Semi-Supervised Classification with Graph Convolutional Networks

**核心公式**:
```
H^{(l+1)} = σ(D^{-1/2} Ã D^{-1/2} H^{(l)} W^{(l)})

其中:
- Ã = A + I (添加自环的邻接矩阵)
- D 是度矩阵，D_ii = Σ_j Ã_ij
- H^{(l)} 是第 l 层的节点表示
- W^{(l)} 是可学习权重矩阵
- σ 是激活函数 (ReLU)
```

**直观理解**:
1. **添加自环**: 每个节点也考虑自己的特征
2. **归一化**: D^{-1/2} Ã D^{-1/2} 确保数值稳定
3. **聚合**: 每个节点聚合邻居的特征（加权平均）
4. **变换**: 通过权重矩阵 W 进行线性变换
5. **激活**: 引入非线性

**单层 GCN 的 Python 实现**:
```python
def graph_convolution(node_features, adj_matrix, weights):
    # 添加自环
    adj_with_self = adj_matrix + np.eye(n_nodes)
    
    # 计算度矩阵
    degrees = np.sum(adj_with_self, axis=1)
    d_inv_sqrt = np.diag(1.0 / np.sqrt(degrees))
    
    # 归一化邻接矩阵
    norm_adj = d_inv_sqrt @ adj_with_self @ d_inv_sqrt
    
    # 图卷积
    output = norm_adj @ node_features @ weights
    
    # ReLU 激活
    output = np.maximum(0, output)
    
    return output
```

#### 图注意力网络 (Graph Attention Network, GAT)

**论文**: Veličković et al. (2018). Graph Attention Networks

**核心思想**: 使用注意力机制学习邻居节点的重要性权重

**注意力系数计算**:
```
α_ij = exp(LeakyReLU(a^T [Wh_i || Wh_j])) / Σ_k∈N(i) exp(LeakyReLU(a^T [Wh_i || Wh_k]))

其中:
- h_i, h_j: 节点 i 和 j 的特征
- W: 可学习权重矩阵
- a: 可学习注意力向量
- ||: 拼接操作
- N(i): 节点 i 的邻居集合
```

**多头注意力 (Multi-Head Attention)**:
- 使用 K 个独立的注意力头
- 每个头学习不同的表示子空间
- 输出拼接或平均

**优势**:
- 自适应学习邻居重要性（不是均匀聚合）
- 可解释性强（注意力权重可视化）
- 适用于异构图（不同类型的邻居）

#### 消息传递神经网络 (Message Passing Neural Network, MPNN)

**通用框架**: Gilmer et al. (2017). Neural Message Passing for Quantum Chemistry

**消息传递阶段**:
```
for t = 1 to T:
    # 消息函数
    m_{i←j} = M_t(h_i, h_j, e_ij)  # 从 j 到 i 的消息
    
    # 聚合函数
    m_i = Σ_{j∈N(i)} m_{i←j}  # 聚合所有邻居消息
    
    # 更新函数
    h_i^{(t+1)} = U_t(h_i^{(t)}, m_i)  # 更新节点表示
```

**读出阶段**:
```
y = R({h_i^{(T)} | i ∈ G})  # 从节点表示得到图级输出
```

**统一视角**: GCN、GAT 都是 MPNN 的特例

## 🔀 2.6 混合膜算法 (Memetic Algorithm) ⭐ NEW

**核心思想**：结合遗传算法的全局探索能力 + 局部搜索的开发能力，实现"探索 - 开发"平衡

**灵感**：文化进化理论 —— 基因进化 + 文化传播（个体学习）

### 算法框架

```
初始化种群
while 未满足终止条件:
    评估种群适应度
    
    # 精英保留 + 局部搜索
    for 每个精英个体:
        if 随机概率 < 局部搜索概率:
            执行局部搜索 (VNS/TS)
            替换原个体
    
    # 遗传操作
    while 新种群未满:
        选择父代 (锦标赛)
        交叉 (OX/PMX)
        变异 (交换/插入)
        添加到新种群
    
    更新种群

返回全局最优解
```

### 关键组件

#### 1. 全局搜索：遗传算法

**作用**：探索解空间，发现潜在优质区域

**操作**：
- **选择**：锦标赛选择 (k=3)
- **交叉**：顺序交叉 (OX) - 保持路径合法性
- **变异**：交换变异 - 引入多样性

**参数**：
- 种群大小：80-150
- 交叉率：0.7-0.9
- 变异率：0.1-0.2
- 精英比例：10-20%

#### 2. 局部搜索：变邻域搜索 (VNS)

**作用**：在精英个体周围深度搜索，提升解质量

**邻域结构**：
- N1: 交换相邻节点
- N2: 交换任意两节点
- N3: 插入操作
- N4: 逆转子序列
- N5: 2-opt

**策略**：
- 仅对精英个体应用（节省计算）
- 概率性应用（避免过早收敛）
- 迭代次数：30-100

### 探索 - 开发平衡

| 组件 | 作用 | 强度 | 计算成本 |
|------|------|------|----------|
| GA 交叉 | 探索新区域 | 高 | 低 |
| GA 变异 | 维持多样性 | 中 | 低 |
| 局部搜索 | 深度优化 | 高 | 高 |

**平衡策略**：
- 局部搜索概率：0.2-0.5（太低=纯 GA，太高=计算浪费）
- 仅应用于精英个体（投资优质解）
- 自适应调整（早期少，后期多）

### 参数调优

**关键参数**：

| 参数 | 推荐范围 | 影响 |

## 2.18 多目标优化 - NSGA-II

**学习日期**: 2026-03-12  
**所属周次**: Week 3 Day 12

### 核心思想

多目标优化处理**多个相互冲突的目标**，寻找 Pareto 最优解集而非单一最优解。

**线缆布线中的多目标问题**:
- 目标 1：最小化成本（线缆长度）
- 目标 2：最小化风险（最大化可靠性）

这两个目标通常冲突：最短路径可能经过高风险区域，而避开风险会增加长度。

### Pareto 最优概念

**支配关系**: 解 A 支配解 B 当且仅当：
1. A 的所有目标都不差于 B
2. A 至少有一个目标严格优于 B

**Pareto 最优解**: 不被任何其他解支配的解

**Pareto 前沿**: 所有 Pareto 最优解在目标空间的集合

```
目标 2 (风险)
    ↑
    |    ·  ·  ·
    |   ·  Pareto 前沿
    |  ·  (非支配解集)
    | ·
    |·
    +----------------→ 目标 1 (成本)
    
    · = 被支配解
    · = Pareto 最优解
```

### NSGA-II 算法流程

**快速非支配排序 (Fast Non-dominated Sorting)**:
```
1. 计算每个个体的支配计数和被支配集合
2. 支配计数为 0 的个体放入第 1 层 (F1)
3. 对 F1 中每个个体，减少其支配个体的计数
4. 计数变为 0 的个体放入 F2
5. 重复直到所有个体分层
```

**拥挤度距离 (Crowding Distance)**:
```
目的：保持解的多样性，避免聚集

计算步骤:
1. 对每个目标，按该目标值排序
2. 边界点拥挤度设为无穷大
3. 中间点拥挤度 = 相邻两点目标值差 / 目标范围

拥挤度大 = 周围解稀疏 = 更值得保留
```

**选择机制**:
```
锦标赛选择:
- 随机选择 k 个个体
- 优先选择等级 (rank) 好的
- 等级相同时选择拥挤度大的
```

**精英保留**:
```
1. 合并父代和子代 (2N 个个体)
2. 快速非支配排序
3. 从 F1 开始依次加入新种群
4. 当某层无法全部加入时，按拥挤度选择
```

### 算法伪代码

```
初始化种群 P0 (大小 N)
评估所有个体

for t = 0 to max_generations:
    # 快速非支配排序
    fronts = fast_non_dominated_sort(Pt)
    
    # 计算拥挤度
    for front in fronts:
        calculate_crowding_distance(front)
    
    # 选择、交叉、变异生成子代 Qt
    Qt = []
    while len(Qt) < N:
        p1 = tournament_selection(Pt)
        p2 = tournament_selection(Pt)
        child = crossover(p1, p2)
        child = mutate(child)
        Qt.append(child)
    
    评估 Qt
    
    # 精英保留
    Rt = Pt ∪ Qt  # 合并，大小 2N
    fronts = fast_non_dominated_sort(Rt)
    
    Pt+1 = []
    i = 0
    while len(Pt+1) + |Fi| <= N:
        Pt+1 = Pt+1 ∪ Fi
        i = i + 1
    
    # 处理最后一层
    Fi = fronts[i]
    calculate_crowding_distance(Fi)
    Fi.sort(by=crowding_distance, descending)
    Pt+1 = Pt+1 ∪ Fi[:(N - len(Pt+1))]

返回 F1 (Pareto 前沿)
```

| 交叉率 | 0.8-0.95 | 高交叉率促进探索 |
| 变异率 | 0.05-0.2 | 维持多样性 |
| 锦标赛大小 | 2-5 | 影响选择压力 |

### 代码实现要点

**1. 支配关系判断**:
```python
def dominates(ind1, ind2):
    not_worse = all(o1 <= o2 for o1, o2 in zip(ind1.obj, ind2.obj))
    strictly_better = any(o1 < o2 for o1, o2 in zip(ind1.obj, ind2.obj))
    return not_worse and strictly_better
```

**2. 拥挤度计算**:
```python
# 对每个目标分别计算
for m in range(num_objectives):
    sorted_front = sorted(front, key=lambda ind: ind.objectives[m])
    sorted_front[0].crowding = float('inf')
    sorted_front[-1].crowding = float('inf')
    
    obj_range = sorted_front[-1].objectives[m] - sorted_front[0].objectives[m]
    for i in range(1, len(sorted_front) - 1):
        sorted_front[i].crowding += (
            (sorted_front[i+1].objectives[m] - sorted_front[i-1].objectives[m]) / obj_range
        )
```

**3. 精英保留选择**:
```python
# 按层加入，最后一层按拥挤度排序
new_pop = []
for front in fronts:
    if len(new_pop) + len(front) <= N:
        new_pop.extend(front)
    else:
        front.sort(key=lambda ind: -ind.crowding_distance)
        new_pop.extend(front[:(N - len(new_pop))])
        break
```

### 2.19 大规模问题求解技巧 - 分解算法

**学习日期**: 2026-03-13  
**所属周次**: Week 3 Day 13

### 核心挑战

当节点数 > 50 时，传统算法面临挑战：
- **计算复杂度爆炸**: TSP 类问题是 NP-hard，O(n!) 或 O(2^n)
- **内存限制**: 距离矩阵 O(n²) 占用大量内存
- **收敛速度慢**: 需要更多迭代才能找到好解
- **局部最优陷阱**: 解空间太大，容易陷入局部最优

### 分解算法思想

**分治法 (Divide and Conquer)**:
```
1. 分解 (Divide): 将大规模问题分解为小规模子问题
2. 求解 (Conquer): 独立求解每个子问题
3. 合并 (Combine): 合并子问题解为全局解
```

**优势**:
- 降低问题复杂度：O(n!) → O((n/k)! × k)
- 可并行计算：子问题独立求解
- 内存友好：只需存储子问题矩阵
- 快速获得可接受解

### 算法流程

```
输入：大规模布线问题 (n > 50 节点)

Step 1: 节点聚类
  - 使用 K-Means 将节点分为 k 个簇
  - 确保起点和终点在合适的簇中
  - 时间复杂度：O(n × k × iterations)

Step 2: 构建簇间连接图
  - 计算簇中心距离
  - 识别每个簇的边界节点
  - 构建簇间邻接图

Step 3: 簇间路径优化
  - 在簇级别求解 TSP
  - 确定簇的访问顺序
  - 使用最近邻或更高级算法

Step 4: 簇内路径优化
  - 对每个簇独立优化
  - 使用 GA/2-opt/VNS 等
  - 可并行执行

Step 5: 合并路径
  - 按簇顺序连接簇内路径
  - 确保边界节点正确连接

Step 6: 全局优化
  - 对整个路径进行 2-opt 改进
  - 跨簇边界优化
  - 迭代改进直到收敛

输出：完整布线路径
```

### 关键技术

#### 1. K-Means 聚类

**目标**: 将空间上接近的节点分到同一簇

**算法**:
```python
from sklearn.cluster import KMeans

kmeans = KMeans(n_clusters=k, random_state=seed, n_init=10)
labels = kmeans.fit_predict(coordinates)
```

**簇数量选择**:
- 经验法则：k ≈ √(n/2)
- 100 节点 → k ≈ 7-10
- 200 节点 → k ≈ 10-14
- 500 节点 → k ≈ 15-20

**注意事项**:
- 确保起点和终点不被孤立
- 簇大小应相对均衡
- 考虑使用层次聚类处理非球形簇

#### 2. 边界节点识别

**目标**: 找到每个簇最适合连接到其他簇的节点

**方法**:
```python
# 簇 i 离簇 j 最近的节点
for node in cluster_i.nodes:
    dist = distance(node, cluster_j.center)
    if dist < min_dist:
        boundary_node = node
```

**应用**:
- 确定簇的入口节点（从前一个簇进入）
- 确定簇的出口节点（到下一个簇离开）

#### 3. 层次优化策略

**两层优化**:
```
Level 1 (宏观): 簇间路径优化
  - 问题规模：k 个簇 (k << n)
  - 算法：最近邻/GA/精确算法
  - 目标：最小化簇间距离

Level 2 (微观): 簇内路径优化
  - 问题规模：n/k 个节点
  - 算法：GA + 2-opt/VNS
  - 目标：最小化簇内路径
```

**优势**:
- 宏观层快速确定大致方向
- 微观层精细优化局部路径
- 两层协同获得全局好解

#### 4. 全局优化改进

**跨簇 2-opt**:
```python
# 允许跨簇边交换
for i in range(1, len(path) - 2):
    for j in range(i + 1, len(path) - 1):
        # 即使 i 和 j 在不同簇也尝试交换
        if improves_solution(path, i, j):
            perform_2opt(path, i, j)
```

**迭代改进**:
- 初始解：分解算法结果
- 改进：随机 2-opt + 接受改进
- 终止：无改进或达到迭代次数

### 复杂度分析

| 步骤 | 时间复杂度 | 说明 |
|------|-----------|------|
| K-Means 聚类 | O(n × k × iter) | 通常 iter=10-20 |
| 簇间图构建 | O(k² × n/k) = O(nk) | 计算边界节点 |
| 簇间 TSP | O(k²) 或 O(k log k) | k << n |
| 簇内优化 | k × O((n/k)²) = O(n²/k) | 可并行 |
| 全局优化 | O(n² × iterations) | 可选步骤 |
| **总计** | **O(n²/k + nk)** | 比 O(n!) 好得多 |

**加速比** (相比直接求解):
- 100 节点，k=10: 约 100-1000 倍
- 200 节点，k=14: 约 1000-10000 倍
- 500 节点，k=16: 约 10000+ 倍

| 方法 | 适用规模 | 解质量 | 计算时间 | 实现难度 |

| 元启发式 (GA/VNS) | 30-100 节点 | ⭐⭐⭐⭐ | 中等 | 中等 |
| **分解算法** | **50-500+ 节点** | **⭐⭐⭐** | **短** | **中等** |
| 贪心算法 | 任意规模 | ⭐⭐ | 极短 | 简单 |
| 深度学习 (GNN) | 任意规模 | ⭐⭐⭐ | 训练长/推理短 | 难 |

| 参数 | 推荐范围 | 影响 |
|------|---------|------|
| 簇数量 k | √(n/2) ± 3 | 太少=子问题大，太多=簇间复杂 |

### 优缺点分析

**优点**:
- ✅ 可处理大规模问题 (100-500+ 节点)
- ✅ 计算时间短 (秒级 vs 分钟/小时级)
- ✅ 可并行化 (簇内独立优化)
- ✅ 内存友好 (小矩阵)
- ✅ 解质量可接受 (通常优于贪心)

**缺点**:
- ❌ 解质量低于直接优化 (分解损失)
- ❌ 簇数量选择敏感
- ❌ 边界节点选择影响大
- ❌ 可能错过跨簇的更优连接

### 改进方向

**1. 自适应聚类**:
- 根据节点密度调整簇大小
- 动态调整簇数量
- 考虑障碍物/风险区域

**2. 重叠簇**:
- 允许节点属于多个簇
- 增加连接灵活性
- 更复杂的合并策略

**3. 多水平分解**:
- 递归分解：簇 → 子簇 → ...
- 适用于超大规模 (1000+ 节点)

**4. 混合方法**:
- 分解 + 深度学习 (GNN 指导聚类)
- 分解 + 强化学习 (学习合并策略)

```python
def two_opt_improve(path, dist_matrix):
    improved = True
    while improved:
        improved = False
        for i in range(1, len(path) - 2):
            for j in range(i + 1, len(path) - 1):
                old_cost = dist[path[i-1], path[i]] + dist[path[j], path[j+1]]
                new_cost = dist[path[i-1], path[j]] + dist[path[i], path[j+1]]

                    improved = True
    return path
```

### 10.1 海上风电场集电系统优化

**案例背景**:
- 海上风电场包含多个风力涡轮机 (WTG)
- 需要设计集电系统电缆网络将所有 WTG 连接到升压站 (OSS)
- 目标：最小化电缆总成本，同时满足电气约束

**问题特点**:
1. **树状拓扑**: 辐射状结构，从 OSS 向外延伸
2. **功率流约束**: 越靠近 OSS 的电缆需要传输更大功率
3. **电缆选型**: 根据功率流选择不同截面的电缆
4. **海况影响**: 水深、海床类型影响安装成本

**数学模型**:
```
min Σ (cable_cost_per_km × length_ij × x_ij)

s.t.
  1. 树状结构约束 (无环、连通)
  2. 功率平衡：流入 = 流出 + 本地发电
  3. 电缆容量：power_ij ≤ max_power_cable
  4. 辐射状运行：每个 WTG 只有一条路径到 OSS
```

**求解方法**:
1. **初始拓扑**: 最小生成树 (Prim 算法)
2. **电缆选型**: 基于功率流的贪心选择
3. **拓扑优化**: 变邻域搜索 (VNS)
4. **成本计算**: 累加各段电缆成本

**算法流程**:
```python
# 1. 构建完全图

topology = nx.minimum_spanning_tree(graph)

# 3. 计算功率流 (从叶子到根)
for node in reverse_topological_order:
    power_flow[node] = sum(power_flow[children]) + turbine_capacity[node]

# 4. 电缆选型
for edge in topology.edges():
    cable = select_cable_for_power(power_flow[edge])
    
# 5. VNS 优化拓扑
for iteration in range(max_iter):
    neighbor = perturb_topology(topology)
    if evaluate(neighbor) < evaluate(topology):
        topology = neighbor

# 6. 输出结果
total_cost = sum(cable.cost * length for all edges)
```

**问题特点**:

3. **MST 初始拓扑**

4. **VNS 拓扑优化**

   ```python
   # 从负荷向电源累加功率
   ```

### 10.3 数据中心网络布线案例

**案例背景**:
大型数据中心网络拓扑设计与电缆布线优化，涉及高密度机架布局、高速网络连接、散热约束和冗余设计。

**问题特点**:
- 高密度机架：48+ 服务器机架，8+ 网络机架
- 高速网络：Spine-Leaf 架构，40Gbps-400Gbps 带宽
- 散热约束：冷热通道设计，冷却区域容量限制
- 冗余要求：N+1 冗余，双路供电/网络
- 多目标：最小化成本 + 最小化延迟 + 最大化可靠性

**核心算法**:

1. **Spine-Leaf 网络架构**
   ```
   Server → ToR → Leaf → Spine → Leaf → ToR → Server
   ```
   - 全互联拓扑：每个 Leaf 连接所有 Spine
   - 任意两点间跳数相同 (通常 2 跳)
   - 高带宽、低延迟、易扩展

2. **3D 空间建模**
   - 机架位置：(行，列，高度 U)
   - 电缆长度计算：水平距离 + 垂直距离 × 1.2 (走线槽余量)
   - 冷却区域：多边形边界，容量限制

3. **电缆类型选择**
   | 类型 | 带宽 | 成本 (元/米) | 延迟 (ns/m) | 适用场景 |
   |------|------|-------------|------------|----------|
   | Cat6 | 1Gbps | 2.0 | 5.0 | 服务器接入 |
   | Cat6A | 10Gbps | 3.5 | 5.0 | ToR 连接 |
   | Fiber OM3 | 40Gbps | 8.0 | 5.0 | 短距离骨干 |
   | Fiber OM4 | 100Gbps | 12.0 | 5.0 | Leaf-Spine |
   | Fiber OS2 | 400Gbps+ | 20.0 | 5.0 | 长距离骨干 |

4. **MST 路由优化 (Kruskal)**
   - 构建所有可能的边
   - 按成本排序
   - 并查集实现连通性检查
   - 生成最小成本树

5. **冗余路径设计**
   - 每个服务器机架连接 2 个 Leaf (主备)
   - Spine-Leaf 全互联天然冗余
   - 冗余连接标记与成本分析

6. **冷却约束检查**
   ```python
   def check_cooling_constraint(rack, zone):
       if rack.in_zone(zone):
           return zone.available_capacity >= rack.heat_output
       return True
   ```

7. **网络性能分析**
   - 总延迟 = Σ(电缆长度 × 5ns/m)
   - 瓶颈带宽 = min(路径上所有电缆带宽)
   - 冗余度 = 冗余连接数 / 总连接数

**数据中心参数** (示例):
- 规模：10 行 × 8 列 = 80 机架位
- 服务器机架：48 个 (8kW/个，散热 6kW)
- 网络机架：12 个 (Spine 4 + Leaf 8)
- 冷却区域：3 个 (冷通道 2 + 热通道 1)
- 总电缆：128 条
- 总长度：~609 米
- 总成本：~3,280 元
- 平均延迟：~24ns

**优化效果**:
- Spine-Leaf 架构：全互联，任意两点 2 跳
- MST 优化：比随机布线节省 ~35% 成本
- 冗余设计：62.5% 连接为冗余 (80/128)
- 冷却利用率：~75% (安全余量 25%)

**关键洞察**:
1. Spine-Leaf 架构是数据中心网络的标准选择
2. 电缆类型选择需权衡带宽需求与成本
3. 冷热通道设计对散热效率至关重要
4. 冗余设计增加成本但大幅提升可靠性
5. 3D 布局可视化帮助发现潜在问题

**可视化输出**:
- `datacenter_layout.png` - 2D 布局图 (机架 + 冷却区 + 电缆)
- `datacenter_topology.png` - Spine-Leaf 网络拓扑图
- `datacenter_cost_analysis.png` - 成本分析 (类型分布/冗余占比/长度分布/冷却利用率)

## 🚗 六、交通网络布线案例 (Day 23)

### 问题背景

城市交通监控与通信网络布线是智慧城市建设的核心基础设施。需要在现有道路网络沿线上部署摄像头和通信节点，同时满足：
- 最小化布线成本
- 保证所有摄像头连接到汇聚节点
- 考虑施工难度和交通影响
- 支持未来扩展

### 数学模型

**目标函数**:
```
min Σ(cable_cost × length × construction_difficulty × traffic_impact)
```

**约束条件**:
1. **连通性约束**: 所有摄像头必须连接到至少一个汇聚节点
2. **容量约束**: 每个汇聚节点连接的摄像头数不超过其容量
3. **路径约束**: 电缆必须沿道路网络铺设
4. **优先级约束**: 高优先级摄像头优先保证连接

### 关键建模要素

1. **道路网络建模**
   - 网格状路口节点 (Intersection)
   - 道路段 (RoadSegment) 连接相邻路口
   - 道路类型：高速公路/主干道/次干道/支路
   - 施工难度系数：0.8 (支路) ~ 2.5 (高速公路)

2. **交通流量影响**
   ```
   traffic_flow = 2.0 - 1.5 × (dist_from_center / max_dist)
   construction_difficulty *= traffic_flow
   ```
   - 中心区域交通流量高，施工难度大
   - 边缘区域施工相对容易

3. **摄像头部署策略**
   - 基于交通流量概率放置 (流量越高越需要监控)
   - 摄像头类型：HD (60%) / 4K (30%) / PTZ (10%)
   - 优先级：1-5 (5 为最高)

4. **汇聚节点布局**
   - 位于 hub 类型的路口
   - 容量：8-15 个摄像头
   - 确保至少有一个汇聚节点

### 优化算法

1. **MST 主干网络优化**
   - 使用 Kruskal/Prim 算法构建最小生成树
   - 边权重 = 电缆成本 × 长度 × 施工难度
   - 形成低成本的主干通信网络

2. **摄像头 - 汇聚节点分配**
   - **最近邻策略**: 分配到地理距离最近的汇聚节点
   - **负载均衡策略**: 考虑汇聚节点容量，避免过载
   - 优先级调度：高优先级摄像头优先分配

3. **VNS 局部优化**
   - 从 MST 解开始
   - 邻域操作：边替换 (保持连通性)
   - 接受准则：改进必接受，劣解概率接受
   - 典型改进：3-8%

### 算法流程

```python
# 1. 创建交通网络
network = TransportationNetwork(grid_size=8, seed=42)

# 2. MST 优化主干网络
optimizer.optimize_with_mst(cable_type="standard")

# 3. 摄像头分配

# 4. VNS 局部优化

# 5. 可视化

```

## ⚡ 性能优化技术 (2026-03-24)

### 性能分析工具

#### cProfile 性能剖析

```python
import cProfile
import pstats

profiler = cProfile.Profile()
profiler.enable()

# 运行待分析代码
your_function()

profiler.disable()
stats = pstats.Stats(profiler).sort_stats('cumulative')
stats.print_stats(20)  # 显示 top 20 耗时函数
```

**关键指标**:
- `cc`: 原始调用次数
- `nc`: 总调用次数（含递归）
- `tt`: 函数内部总时间（不含子调用）
- `ct`: 累计时间（含子调用）
- `time_per_call`: tt/nc

**热点识别**:
1. 按 `cumulative` 排序：找最耗时的调用链
2. 按 `time` 排序：找自身耗时最长的函数
3. 关注高频调用的小函数（累积效应）

### 加速技术

#### 1. Numba JIT 编译

**原理**: 即时编译 Python 代码为机器码

**安装**: `pip install numba`

**基本用法**:
```python
from numba import jit, prange

@jit(nopython=True, cache=True)
def fast_function(x, y):
    result = 0.0
    for i in range(len(x)):
        result += x[i] * y[i]
    return result
```

**关键参数**:
- `nopython=True`: 纯机器码模式（最快）
- `cache=True`: 缓存编译结果
- `parallel=True`: 自动并行化循环
- `prange`: 并行 range（替代 range）

**加速效果**:
- 数值计算：10-100 倍
- 循环密集型：50-500 倍
- NumPy 操作：2-10 倍

**注意事项**:
- 避免 Python 对象（list/dict）
- 使用 NumPy 数组
- 类型推断失败会回退到 Python 模式

---

#### 2. 多进程并行化

**适用场景**: CPU 密集型任务，多起点搜索

**multiprocessing 示例**:
```python
from multiprocessing import Pool, cpu_count

def worker(args):
    # 独立计算任务
    return result

if __name__ == "__main__":
    n_workers = cpu_count()
    
    with Pool(processes=n_workers) as pool:
        results = pool.map(worker, task_list)
```

**ProcessPoolExecutor (推荐)**:
```python
from concurrent.futures import ProcessPoolExecutor

with ProcessPoolExecutor(max_workers=8) as executor:
    results = list(executor.map(func, data_list))
```

**加速效果**:
- 4 核 CPU: ~3-4 倍
- 8 核 CPU: ~6-8 倍
- 受 Amdahl 定律限制（串行部分）

**注意事项**:

- 使用 `if __name__ == "__main__"` 保护

#### 2. 数据结构优化

```python
# 预计算所有点对距离
dist_matrix = np.zeros((n, n))
for i in range(n):
    for j in range(i+1, n):
        d = calculate_distance(points[i], points[j])
        dist_matrix[i, j] = dist_matrix[j, i] = d
```

```python
result = np.dot(x, y)
```

### 8.4 关键实现

**管廊优惠逻辑**:
```python
def _segment_near_corridor(p1, p2, corridor):
    # 计算线段中点到管廊线段的距离
    mid = ((p1[0]+p2[0])/2, (p1[1]+p2[1])/2)
    dist = point_to_segment_distance(mid, corridor.start, corridor.end)
    return dist < 30  # 30 米内可使用管廊
```

**障碍物穿越检测**:
```python
def contains_segment(self, p1, p2):
    # 射线法判断线段中点是否在多边形内
    mid = ((p1[0]+p2[0])/2, (p1[1]+p2[1])/2)
    return self._point_in_polygon(mid)
```

**Kruskal 算法**:
```python
# Union-Find 数据结构
parent = {node_id: node_id for node_id in valid_nodes}

def find(x):
    if parent[x] != x:
        parent[x] = find(parent[x])  # 路径压缩
    return parent[x]

def union(x, y):
    px, py = find(x), find(y)
    if px == py: return False
    # 按秩合并
    parent[py] = px
    return True
```

### 9.3 参数调优经验法则

#### 遗传算法 (GA)
```
关键参数调优顺序：population_size → crossover_rate → mutation_rate

经验值:
- population_size: 问题规模的 5-10 倍 (50-500)
- crossover_rate: 0.7-0.9 (高交叉率促进探索)
- mutation_rate: 0.05-0.2 (低变异率保持稳定性)
- generations: 200-1000 (根据收敛情况调整)

常见问题解决:
- 早熟收敛 → 增加变异率，增加种群多样性
- 收敛太慢 → 增加交叉率，减少种群大小
- 解质量差 → 增加迭代次数，改进选择策略
```

#### 粒子群优化 (PSO)
```
关键参数调优顺序：w → c1/c2 → swarm_size

经验值:
- w (惯性权重): 0.4-0.9 (线性递减策略：0.9→0.4)
- c1 (个体认知): 1.5-2.0
- c2 (社会认知): 1.5-2.0 (通常 c1=c2)
- swarm_size: 20-50 (PSO 对小种群也有效)
- iterations: 100-500

常见问题解决:
- 陷入局部最优 → 增加 w 或采用自适应 w 策略
- 收敛过快 → 减小 c1/c2，增加随机性
```

#### 模拟退火 (SA)
```
关键参数调优顺序：initial_temp → cooling_rate → min_temp

经验值:
- initial_temp: 使初始接受率约 80% (试错法确定)
- cooling_rate: 0.85-0.99 (慢冷却质量好但耗时)
- min_temp: 1e-3 到 1e-5
- iterations_per_temp: 10-100

常见问题解决:
- 接受率太低 → 提高初始温度
- 搜索太慢 → 增加冷却率
```

#### 变邻域搜索 (VNS)
```
关键参数调优顺序：neighborhood 设计 → 迭代次数

经验值:
- 邻域数量：3-5 个 (从简单到复杂)
- 邻域顺序：先小扰动后大扰动
- 每个邻域迭代：10-50 次
- 常用邻域：swap, insert, reverse, 2-opt

常见问题解决:
- 改进有限 → 设计更有针对性的邻域结构
- 运行时间长 → 减少邻域数量或迭代次数
```

## 📡 十一、案例研究四：电信网络优化

### 11.1 问题背景

电信网络是现代基础设施的核心，光纤网络设计需要考虑：

1. **多层网络架构**：
   - 核心层 (Core)：高速骨干网，连接主要数据中心
   - 汇聚层 (Distribution)：区域汇聚，连接核心与接入
   - 接入层 (Access)：用户接入，最后一公里

2. **可靠性要求**：
   - 自愈网络：故障时自动切换备用路径
   - 双归属：关键节点连接两个上级节点
   - 环形拓扑：核心层采用双环结构

3. **服务质量 (QoS)**：
   - 延迟约束：端到端延迟 < 50ms
   - 带宽保障：满足峰值流量需求
   - 抖动控制：延迟变化 < 5ms

### 11.2 数学模型

**网络表示**：`G = (V, E)`
- `V`: 节点集合 (核心/汇聚/接入)
- `E`: 链路集合 (光纤电缆)

**决策变量**：
- `y_e ∈ {0,1}`: 是否建设链路 e
- `x_e^k`: 流量 k 在链路 e 上的分配
- `r_e ∈ {primary, backup}`: 链路角色

**目标函数**：
```
min Σ c_e * y_e + α * Σ latency_k
```
- 建设成本最小化
- 延迟加权 (α 为权重系数)

**约束条件**：
1. **流量守恒**：
   ```
   Σ x_e^k (流入) - Σ x_e^k (流出) = demand_k
   ```

2. **容量限制**：
   ```
   Σ x_e^k ≤ capacity_e * y_e
   ```

3. **延迟约束**：
   ```
   Σ latency_e (路径) ≤ max_latency_k
   ```

4. **连通性**：
   ```
   每个接入节点至少有 2 条路径到核心层
   ```

5. **环形拓扑**：
   ```
   核心节点形成闭合环
   ```

```python
    """
    基于最短路径的流量分配
    
    1. 构建带权图 (权重=延迟)
    2. 对每个需求运行 Dijkstra
    3. 检查带宽约束
    4. 分配容量

    for demand in demands:
        # 最短路径 (延迟最小)
        path = shortest_path(G, demand.source, demand.destination)
        
        # 检查带宽约束
        min_capacity = min(link.capacity - link.used for link in path)
        
        if min_capacity >= demand.bandwidth:
            # 分配带宽
            for link in path:
                link.used_capacity += demand.bandwidth
        else:
            # 需求阻塞
            blocked_demands += 1
```

## 🚀 十二、性能优化技术

_主题：Numba JIT 编译与并行化优化_

### 12.1 性能优化概述

在大规模网络优化问题中，计算效率是关键瓶颈。本章介绍两种核心优化技术：

1. **JIT (Just-In-Time) 编译**：Numba 将 Python 代码编译为机器码
2. **并行化**：利用多核 CPU 同时执行独立任务

**优化目标**：
- 减少计算时间
- 提高可扩展性
- 保持代码可读性

### 12.2 Numba JIT 编译原理

#### 什么是 Numba？

Numba 是 NumPy 感知的 Python JIT 编译器，可将 Python 函数编译为优化的机器码。

**工作流程**：
```
Python 源代码
    ↓
Numba 装饰器 (@jit)
    ↓
LLVM IR (中间表示)
    ↓
机器码 (CPU/GPU)
    ↓
执行 (接近 C/C++ 速度)
```

#### 核心装饰器

```python
from numba import jit, prange

# 基础 JIT 编译
@jit(nopython=True, cache=True)
def fast_function(x, y):
    return x * y + np.sqrt(x)

# 并行化 (需要 parallel=True)
@jit(nopython=True, parallel=True)
def parallel_loop(arr):
    result = np.zeros(len(arr))
    for i in prange(len(arr)):  # prange 支持并行
        result[i] = arr[i] ** 2
    return result
```

#### 关键参数

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| `nopython=True` | 强制 nopython 模式（最快） | ✅ 总是使用 |
| `cache=True` | 缓存编译结果 | ✅ 推荐 |
| `parallel=True` | 启用自动并行化 | 视情况 |
| `fastmath=True` | 放宽浮点精度（更快） | 谨慎使用 |

#### 优化前：纯 Python 实现

```python
def calculate_distance_matrix_python(x_coords, y_coords):
    n = len(x_coords)
    dist_matrix = np.zeros((n, n))
    for i in range(n):
        for j in range(i + 1, n):
            dist = calculate_distance_python(x_coords[i], y_coords[i],
                                            x_coords[j], y_coords[j])
            dist_matrix[i, j] = dist
            dist_matrix[j, i] = dist
    return dist_matrix
```

#### Dijkstra 算法优化

```python
@jit(nopython=True, cache=True)
def dijkstra_numba(dist_matrix, latency_matrix, source, dest, n_nodes):
    INF = 1e10
    dist = np.full(n_nodes, INF, dtype=np.float64)
    prev = np.full(n_nodes, -1, dtype=np.int64)
    visited = np.zeros(n_nodes, dtype=np.bool_)
    
    dist[source] = 0.0
    
    for _ in range(n_nodes):
        # 找到未访问的最小距离节点
        min_dist = INF
        u = -1
        for i in range(n_nodes):
            if not visited[i] and dist[i] < min_dist:
                min_dist = dist[i]
                u = i
        
        if u == -1 or u == dest:
            break
        
        visited[u] = True
        
        # 更新邻居
        for v in range(n_nodes):
            if not visited[v] and dist_matrix[u, v] > 0:
                alt = dist[u] + latency_matrix[u, v]
                if alt < dist[v]:
                    dist[v] = alt
                    prev[v] = u
    
    return dist, prev
```

### 12.4 并行化技术

#### 多线程 vs 多进程

| 特性 | 多线程 | 多进程 |
|------|--------|--------|
| 内存共享 | ✅ 共享 | ❌ 独立 |
| GIL 限制 | ✅ 受影响 | ❌ 绕过 |
| 适用场景 | I/O 密集 | CPU 密集 |
| 启动开销 | 低 | 高 |

#### 并行路径评估

```python
from concurrent.futures import ProcessPoolExecutor

def evaluate_path(path, latency_matrix):
    """评估单条路径的延迟"""
    total = 0.0
    for i in range(len(path) - 1):
        total += latency_matrix[path[i], path[i+1]]
    return total

def parallel_evaluate_paths(paths, latency_matrix):
    """并行评估多条路径"""
    with ProcessPoolExecutor() as executor:
        futures = [
            executor.submit(evaluate_path, path, latency_matrix)
            for path in paths
        ]
        results = [f.result() for f in futures]
    return results
```

#### Numba 并行循环

```python
from numba import prange

@jit(nopython=True, parallel=True)
def parallel_path_evaluation(paths, latency_matrix, n_paths):
    path_latencies = np.zeros(n_paths)
    for i in prange(n_paths):  # 自动并行
        path = paths[i]
        total_latency = 0.0
        for j in range(len(path) - 1):
            if path[j] >= 0 and path[j+1] >= 0:
                total_latency += latency_matrix[path[j], path[j+1]]
        path_latencies[i] = total_latency
    return path_latencies
```

### 12.6 优化技巧总结

#### ✅ 最佳实践

1. **使用 NumPy 数组**：
   ```python
   # ❌ 慢：Python 列表
   coords = [1.0, 2.0, 3.0]
   
   # ✅ 快：NumPy 数组
   coords = np.array([1.0, 2.0, 3.0], dtype=np.float64)
   ```

2. **避免 Python 对象**：
   ```python
   # ❌ Numba 不支持
   result = {'distance': 10.0, 'path': [1, 2, 3]}
   
   # ✅ 使用 NumPy 结构化数组
   result = np.array([(10.0, [1, 2, 3])], 
                     dtype=[('dist', 'f8'), ('path', 'i4', 3)])
   ```

3. **预热 JIT 编译**：
   ```python
   # 首次运行包含编译时间
   @jit(nopython=True)
   def my_func(x):
       return x * 2
   
   # 预热
   my_func(np.array([1.0]))  # 编译
   
   # 正式使用（快）
   result = my_func(large_array)
   ```

4. **使用 `prange` 并行**：
   ```python
   @jit(nopython=True, parallel=True)
   def parallel_sum(arr):
       total = 0.0
       for i in prange(len(arr)):  # 自动并行
           total += arr[i]
       return total
   ```

#### ⚠️ 常见陷阱

1. **对象模式回退**：
   ```python
   # ❌ 会回退到对象模式（慢）
   @jit
   def bad_func(x):
       return len(x)  # Numba 无法推断类型
   
   # ✅ 指定类型签名
   @jit(nopython=True)
   def good_func(x: np.ndarray) -> int:
       return len(x)
   ```

2. **全局变量**：
   ```python
   # ❌ 全局变量可能导致问题
   GLOBAL_CONST = 10
   
   @jit(nopython=True)
   def func(x):
       return x * GLOBAL_CONST  # 可能失败
   
   # ✅ 作为参数传递
   @jit(nopython=True)
   def func(x, const):
       return x * const
   ```

3. **Python 标准库**：
   ```python
   # ❌ Numba 不支持大多数 Python 标准库
   import random
   x = random.random()
   
   # ✅ 使用 NumPy
   import numpy as np
   x = np.random.random()
   ```
