## 📐 一、数学规划法

### 1.1 MILP (混合整数线性规划)

**核心思想**：将问题建模为线性目标函数 + 线性约束，部分变量为整数

**标准形式**：
```
min c^T x
s.t.  Ax ≤ b
      x ≥ 0
      x_i ∈ Z (部分变量)
```

**布线问题建模**：
- **决策变量**：
  - `x_e`: 边 e 上的流量（连续）
  - `y_e`: 是否使用边 e（0-1 整数）
  
- **目标函数**：最小化总成本 `min Σ c_e * y_e`

- **约束**：
  1. 流量守恒：流入 = 流出（除源汇点）
  2. 容量限制：`x_e ≤ cap_e * y_e`
  3. 整数性：`y_e ∈ {0,1}`

**求解器**：
- Gurobi (商业，最快)
- CPLEX (商业)
- CBC (开源，PuLP 内置)
- SCIP (开源)

**优缺点**：
- ✅ 保证全局最优
- ✅ 理论成熟
- ❌ 计算复杂度高（NP-hard）
- ❌ 大规模问题求解难

### 1.2 最短路径树方法

#### Dijkstra 算法

**核心思想**：贪心策略，每次选择距离最小的未访问节点

**伪代码**：
```
dist[source] = 0, 其他为∞
PQ = 优先队列 [(0, source)]

while PQ 非空:
    (d, u) = PQ.pop()
    if u 已访问：continue
    
    for v in neighbors(u):
        new_dist = dist[u] + weight(u,v)
        if new_dist < dist[v]:
            dist[v] = new_dist
            prev[v] = u
            PQ.push((new_dist, v))
```

**时间复杂度**：O((V+E)logV)

**适用场景**：
- 单源最短路径
- 边权非负
- 需要精确解

#### A* 算法

**改进**：加入启发式函数 h(n)

**评估函数**：`f(n) = g(n) + h(n)`
- `g(n)`: 从起点到 n 的实际代价
- `h(n)`: 从 n 到终点的估计代价（如欧氏距离）

**关键**：启发式函数必须是可采纳的（不高估实际代价）

## 🔥 二、启发式算法

### 2.1 遗传算法 (GA)

**灵感**：生物进化论

**核心操作**：
1. **编码**：路径 → 染色体（排列）
2. **选择**：锦标赛/轮盘赌
3. **交叉**：OX/PMX
4. **变异**：交换/逆转

**参数调优**：
- 种群大小：50-200
- 交叉率：0.7-0.9
- 变异率：0.01-0.2
- 精英保留：5-10%

**优缺点**：
- ✅ 全局搜索能力强
- ✅ 并行性好
- ❌ 收敛速度慢
- ❌ 参数敏感

### 2.2 粒子群优化 (PSO)

**灵感**：鸟群觅食

**速度更新**：
```
v = w*v + c1*r1*(pbest-x) + c2*r2*(gbest-x)
x = x + v
```

- `w`: 惯性权重
- `c1, c2`: 学习因子
- `pbest`: 个体最优
- `gbest`: 全局最优

### 2.3 模拟退火 (SA)

**灵感**：金属退火

**Metropolis 准则**：
```
如果 ΔE < 0: 接受新解
否则：以概率 exp(-ΔE/T) 接受
```

**退火计划**：
- 初始温度：高
- 降温系数：0.9-0.99
- 终止温度：接近 0

### 2.4 变邻域搜索 (VNS)

**核心思想**：系统性地改变邻域结构

**流程**：
```
for k = 1 to k_max:
    扰动：在当前解的第 k 个邻域随机扰动
    局部搜索：在新邻域内搜索
    如果改进：接受并重置 k=1
    否则：k = k+1
```

## 💡 五、实战技巧

### MILP 建模技巧

1. **紧化约束**：尽可能使用紧的上界
2. **对称性破缺**：添加约束减少对称解
3. **Lazy Constraint**：动态添加约束（如子环消除）
4. **Warm Start**：用启发式解初始化

### 启发式调参

1. **多样性 vs 收敛**：变异率平衡
2. **精英保留**：防止退化
3. **自适应参数**：根据搜索状态调整
4. **混合策略**：GA+ 局部搜索

## 📅 Day 2 新增内容 (2026-03-03)

### 2.2 粒子群优化 (PSO) - 详细实现

**算法流程**:
```
1. 初始化粒子群（随机位置和速度）
2. 评估每个粒子的适应度
3. 更新个体最优 (pbest) 和全局最优 (gbest)
4. 更新速度：v = w*v + c1*r1*(pbest-x) + c2*r2*(gbest-x)
5. 更新位置：x = x + v
6. 重复 2-5 直到收敛
```

**关键参数**:
- `w` (惯性权重): 0.4-0.9，平衡全局/局部搜索
- `c1` (个体学习因子): 通常 1.5-2.0
- `c2` (群体学习因子): 通常 1.5-2.0
- 种群大小：30-50

**线缆布线应用**:
- 优化路径控制点坐标
- 连续变量优化
- 可加入障碍物惩罚项

**代码示例**: `examples/04_pso.py`

### 1.3 A* 搜索算法 - 详细实现

**评估函数**: `f(n) = g(n) + h(n)`
- `g(n)`: 从起点到 n 的实际代价
- `h(n)`: 从 n 到终点的启发式估计

**常用启发式**:
| 启发式 | 公式 | 适用场景 |
|--------|------|----------|
| 欧氏距离 | √(dx²+dy²) | 任意方向移动 |
| 曼哈顿距离 | |dx|+|dy| | 四方向移动 |
| 切比雪夫距离 | max(|dx|,|dy|) | 八方向移动 |
| Octile 距离 | max+ (√2-1)*min | 八方向精确估计 |

**可采纳性**: 启发式必须不高估实际代价，才能保证找到最优路径

**搜索效率**:
- 时间复杂度：O(b^d)，b 为分支因子，d 为深度
- 空间复杂度：O(b^d)，需要存储 OpenSet

**线缆布线应用**:
- 网格环境路径规划
- 避开障碍物
- 机房/工厂布局布线

**代码示例**: `examples/06_astar.py`

## 📅 Day 3 新增内容 (2026-03-04)

### 1.4 最小生成树 (Minimum Spanning Tree, MST)

**问题定义**:
在连通加权无向图 G=(V,E) 中，找到一棵包含所有顶点的树 T，使得树上所有边的权重之和最小。

**性质**:
- 包含 V 个顶点和 V-1 条边
- 连通且无环
- 总权重最小

---

#### Prim 算法

**核心思想**: 贪心策略，从单个顶点开始，逐步扩展树

**算法流程**:
```
1. 选择任意起始顶点 s，加入集合 S
2. 维护优先队列 PQ，存储所有连接 S 和 V-S 的边
3. while |S| < |V|:
     从 PQ 中取出权重最小的边 (u, v)，其中 u∈S, v∉S
     将 v 加入 S，边 (u, v) 加入 MST
     将 v 的所有邻边加入 PQ
4. 返回 MST
```

**时间复杂度**:
- 使用二叉堆：O(E log V)
- 使用斐波那契堆：O(E + V log V)

**空间复杂度**: O(V + E)

**适用场景**:
- 稠密图（边多）
- 需要逐步构建 MST
- 适合邻接表/邻接矩阵表示

---

#### Kruskal 算法

**核心思想**: 贪心策略，按边权重排序，依次选择不形成环的边

**算法流程**:
```
1. 将所有边按权重从小到大排序
2. 初始化并查集，每个顶点为独立集合
3. for 每条边 (u, v) in 排序后的边:
     if find(u) != find(v):  # 不在同一集合
       将边 (u, v) 加入 MST
       union(u, v)  # 合并集合
4. 返回 MST
```

**并查集优化**:
- **路径压缩**: find 操作时压缩路径
- **按秩合并**: 小树合并到大树

**时间复杂度**: O(E log E) 或 O(E log V)
- 排序：O(E log E)
- 并查集操作：O(E α(V))，α为反阿克曼函数（几乎为常数）

**空间复杂度**: O(V + E)

**适用场景**:
- 稀疏图（边少）
- 边已排序或易于排序
- 实现简单

---

#### 两种算法对比

| 特性 | Prim | Kruskal |
|------|------|---------|
| 贪心策略 | 顶点扩展 | 边选择 |
| 数据结构 | 优先队列 | 并查集 |
| 时间复杂度 | O(E log V) | O(E log E) |
| 适合图类型 | 稠密图 | 稀疏图 |
| 实现难度 | 中等 | 简单 |
| 是否需要连通 | 是 | 是（可检测） |

#### 线缆布线应用

**场景**: 园区/建筑网络布线

**问题建模**:
- **顶点**: 建筑/设备位置
- **边**: 可能的布线路径
- **权重**: 布线成本（距离 × 单位成本）

**优势**:
- 保证所有节点连通
- 总成本最小
- 无冗余连接（无环）

**实际考虑**:
1. **冗余需求**: MST 无冗余，实际可能需要备份线路
2. **容量限制**: 某些边可能有容量上限
3. **地形因素**: 权重可包含地形难度系数
4. **已有设施**: 部分边可能已存在（权重为 0）

**扩展**:
- **Steiner Tree**: 允许添加中间节点（NP-hard）
- **度约束 MST**: 限制节点最大度数
- **多目标 MST**: 同时优化成本、延迟等

**Kruskal 算法**:
```python
# 并查集实现
class UnionFind:
    def find(self, x):
        if parent[x] != x:
            parent[x] = self.find(parent[x])  # 路径压缩
        return parent[x]
    
    def union(self, x, y):
        px, py = self.find(x), self.find(y)
        if px == py:
            return False
        # 按秩合并
        parent[py] = px
        return True

# 主算法
```
