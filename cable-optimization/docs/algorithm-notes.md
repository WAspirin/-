## 📐 一、数学规划法

### 1.1 MILP (混合整数线性规划)

**核心思想**：将问题建模为线性目标函数 + 线性约束，部分变量为整数

**标准形式**：
```
min c^T x
s.t.  Ax ≤ b
      x ≥ 0
      x_i ∈ Z (部分变量)
```

**布线问题建模**：
- **决策变量**：
  - `x_e`: 边 e 上的流量（连续）
  - `y_e`: 是否使用边 e（0-1 整数）
  
- **目标函数**：最小化总成本 `min Σ c_e * y_e`

- **约束**：
  1. 流量守恒：流入 = 流出（除源汇点）
  2. 容量限制：`x_e ≤ cap_e * y_e`
  3. 整数性：`y_e ∈ {0,1}`

**求解器**：
- Gurobi (商业，最快)
- CPLEX (商业)
- CBC (开源，PuLP 内置)
- SCIP (开源)

**优缺点**：
- ✅ 保证全局最优
- ✅ 理论成熟
- ❌ 计算复杂度高（NP-hard）
- ❌ 大规模问题求解难

### 1.2 最短路径树方法

#### Dijkstra 算法

**核心思想**：贪心策略，每次选择距离最小的未访问节点

**伪代码**：
```
dist[source] = 0, 其他为∞
PQ = 优先队列 [(0, source)]

while PQ 非空:
    (d, u) = PQ.pop()
    if u 已访问：continue
    
    for v in neighbors(u):
        new_dist = dist[u] + weight(u,v)
        if new_dist < dist[v]:
            dist[v] = new_dist
            prev[v] = u
            PQ.push((new_dist, v))
```

**时间复杂度**：O((V+E)logV)

**适用场景**：
- 单源最短路径
- 边权非负
- 需要精确解

#### A* 算法

**改进**：加入启发式函数 h(n)

**评估函数**：`f(n) = g(n) + h(n)`
- `g(n)`: 从起点到 n 的实际代价
- `h(n)`: 从 n 到终点的估计代价（如欧氏距离）

**关键**：启发式函数必须是可采纳的（不高估实际代价）

## 🔥 二、启发式算法

### 2.1 遗传算法 (GA)

**灵感**：生物进化论

**核心操作**：
1. **编码**：路径 → 染色体（排列）
2. **选择**：锦标赛/轮盘赌
3. **交叉**：OX/PMX
4. **变异**：交换/逆转

**参数调优**：
- 种群大小：50-200
- 交叉率：0.7-0.9
- 变异率：0.01-0.2
- 精英保留：5-10%

**优缺点**：
- ✅ 全局搜索能力强
- ✅ 并行性好
- ❌ 收敛速度慢
- ❌ 参数敏感

### 2.2 粒子群优化 (PSO)

**灵感**：鸟群觅食

**速度更新**：
```
v = w*v + c1*r1*(pbest-x) + c2*r2*(gbest-x)
x = x + v
```

- `w`: 惯性权重
- `c1, c2`: 学习因子
- `pbest`: 个体最优
- `gbest`: 全局最优

### 2.3 模拟退火 (SA)

**灵感**：金属退火

**Metropolis 准则**：
```
如果 ΔE < 0: 接受新解
否则：以概率 exp(-ΔE/T) 接受
```

**退火计划**：
- 初始温度：高
- 降温系数：0.9-0.99
- 终止温度：接近 0

### 2.4 变邻域搜索 (VNS)

**核心思想**：系统性地改变邻域结构

**流程**：
```
for k = 1 to k_max:
    扰动：在当前解的第 k 个邻域随机扰动
    局部搜索：在新邻域内搜索
    如果改进：接受并重置 k=1
    否则：k = k+1
```

### 2.5 禁忌搜索 (Tabu Search, TS) ⭐ NEW

**核心思想**：使用禁忌表记录最近访问过的解或操作，避免循环；允许接受劣解以跳出局部最优

**灵感**：人类记忆机制 —— 记住最近去过的地方，避免重复访问

**核心机制**：

1. **禁忌表 (Tabu List)**
   - 记录最近执行的操作或访问的解
   - 固定长度（禁忌长度 tabu_tenure）
   - 先进先出 (FIFO) 更新

2. **藐视准则 (Aspiration Criterion)**
   - 如果新解优于历史最优，即使在禁忌表中也可接受
   - 避免错过优质解

3. **长期记忆 (Frequency Memory)**
   - 记录解或操作的访问频率
   - 用于多样化策略（选择低频操作）

**算法流程**：
```
初始化：生成初始解，清空禁忌表
while 未满足终止条件:
    生成邻域解集合
    选择最佳非禁忌解（或满足藐视准则）
    更新当前解
    将对应操作加入禁忌表
    如果优于历史最优：更新历史最优
    如果长期无改进：执行多样化策略
```

**邻域操作**：
- **Swap**: 交换两个位置
- **Insert**: 插入操作（将一个元素移到另一位置）
- **Reverse**: 逆转子序列
- **2-opt**: 经典 TSP 邻域操作

**参数调优**：
- **禁忌长度**: 问题规模的 10-30% (如 n=50, tenure=5-15)
  - 太小：容易循环
  - 太大：限制搜索空间
- **邻域大小**: 20-50 个采样
- **最大迭代**: 500-2000
- **无改进限制**: 100-300 次

**优缺点**：
- ✅ 避免循环搜索
- ✅ 允许接受劣解，跳出局部最优
- ✅ 灵活性强，可与多种邻域结合
- ❌ 参数敏感（禁忌长度）
- ❌ 需要设计合适的邻域结构

**与 VNS 对比**：
| 特性 | VNS | TS |
|------|-----|----|
| 邻域切换 | 系统性 (k=1→k_max) | 固定邻域集合 |
| 记忆机制 | 无 | 禁忌表 + 长期记忆 |
| 接受准则 | 只接受改进 | 可接受劣解 |
| 避免循环 | 邻域变化 | 禁忌表 |

**适用场景**：
- 组合优化问题（TSP、调度、布线）
- 中等规模问题 (50-200 节点)
- 需要精细搜索的问题

**线缆布线应用**：
```python
# 编码：节点访问顺序的排列
solution = [0, 5, 3, 8, 1, 7, ...]

# 邻域：交换两个节点位置
neighbor = swap(solution, i=2, j=5)

# 禁忌：记录最近交换的位置对
tabu_list.append(('swap', 2, 5))

# 藐视：如果新解成本 < 历史最优，接受
if new_cost < best_cost:
    accept(neighbor)  # 即使在禁忌表中
```

**实现提示**：
1. 使用 deque 实现禁忌表（自动 FIFO）
2. 用 tuple 作为禁忌表键（操作类型 + 参数）
3. 长期记忆用 dict 记录频率
4. 多样化策略：选择低频操作

### 2.2 粒子群优化 (PSO) - 详细实现

**算法流程**:
```
1. 初始化粒子群（随机位置和速度）
2. 评估每个粒子的适应度
3. 更新个体最优 (pbest) 和全局最优 (gbest)
4. 更新速度：v = w*v + c1*r1*(pbest-x) + c2*r2*(gbest-x)
5. 更新位置：x = x + v
6. 重复 2-5 直到收敛
```

**关键参数**:
- `w` (惯性权重): 0.4-0.9，平衡全局/局部搜索
- `c1` (个体学习因子): 通常 1.5-2.0
- `c2` (群体学习因子): 通常 1.5-2.0
- 种群大小：30-50

**线缆布线应用**:
- 优化路径控制点坐标
- 连续变量优化
- 可加入障碍物惩罚项

**代码示例**: `examples/04_pso.py`

### 1.3 A* 搜索算法 - 详细实现

**评估函数**: `f(n) = g(n) + h(n)`
- `g(n)`: 从起点到 n 的实际代价
- `h(n)`: 从 n 到终点的启发式估计

**常用启发式**:
| 启发式 | 公式 | 适用场景 |
|--------|------|----------|
| 欧氏距离 | √(dx²+dy²) | 任意方向移动 |
| 曼哈顿距离 | |dx|+|dy| | 四方向移动 |
| 切比雪夫距离 | max(|dx|,|dy|) | 八方向移动 |
| Octile 距离 | max+ (√2-1)*min | 八方向精确估计 |

**可采纳性**: 启发式必须不高估实际代价，才能保证找到最优路径

**搜索效率**:
- 时间复杂度：O(b^d)，b 为分支因子，d 为深度
- 空间复杂度：O(b^d)，需要存储 OpenSet

**线缆布线应用**:
- 网格环境路径规划
- 避开障碍物
- 机房/工厂布局布线

**代码示例**: `examples/06_astar.py`

## 📅 Day 3 新增内容 (2026-03-04)

### 1.4 最小生成树 (Minimum Spanning Tree, MST)

**问题定义**:
在连通加权无向图 G=(V,E) 中，找到一棵包含所有顶点的树 T，使得树上所有边的权重之和最小。

**性质**:
- 包含 V 个顶点和 V-1 条边
- 连通且无环
- 总权重最小

---

#### Prim 算法

**核心思想**: 贪心策略，从单个顶点开始，逐步扩展树

**算法流程**:
```
1. 选择任意起始顶点 s，加入集合 S
2. 维护优先队列 PQ，存储所有连接 S 和 V-S 的边
3. while |S| < |V|:
     从 PQ 中取出权重最小的边 (u, v)，其中 u∈S, v∉S
     将 v 加入 S，边 (u, v) 加入 MST
     将 v 的所有邻边加入 PQ
4. 返回 MST
```

**时间复杂度**:
- 使用二叉堆：O(E log V)
- 使用斐波那契堆：O(E + V log V)

**空间复杂度**: O(V + E)

**适用场景**:
- 稠密图（边多）
- 需要逐步构建 MST
- 适合邻接表/邻接矩阵表示

---

#### Kruskal 算法

**核心思想**: 贪心策略，按边权重排序，依次选择不形成环的边

**算法流程**:
```
1. 将所有边按权重从小到大排序
2. 初始化并查集，每个顶点为独立集合
3. for 每条边 (u, v) in 排序后的边:
     if find(u) != find(v):  # 不在同一集合
       将边 (u, v) 加入 MST
       union(u, v)  # 合并集合
4. 返回 MST
```

**并查集优化**:
- **路径压缩**: find 操作时压缩路径
- **按秩合并**: 小树合并到大树

**时间复杂度**: O(E log E) 或 O(E log V)
- 排序：O(E log E)
- 并查集操作：O(E α(V))，α为反阿克曼函数（几乎为常数）

**空间复杂度**: O(V + E)

**适用场景**:
- 稀疏图（边少）
- 边已排序或易于排序
- 实现简单

---

#### 两种算法对比

| 特性 | Prim | Kruskal |
|------|------|---------|
| 贪心策略 | 顶点扩展 | 边选择 |
| 数据结构 | 优先队列 | 并查集 |
| 时间复杂度 | O(E log V) | O(E log E) |
| 适合图类型 | 稠密图 | 稀疏图 |
| 实现难度 | 中等 | 简单 |
| 是否需要连通 | 是 | 是（可检测） |

---

#### 线缆布线应用

**场景**: 园区/建筑网络布线

**问题建模**:
- **顶点**: 建筑/设备位置
- **边**: 可能的布线路径
- **权重**: 布线成本（距离 × 单位成本）

**优势**:
- 保证所有节点连通
- 总成本最小
- 无冗余连接（无环）

**实际考虑**:
1. **冗余需求**: MST 无冗余，实际可能需要备份线路
2. **容量限制**: 某些边可能有容量上限
3. **地形因素**: 权重可包含地形难度系数
4. **已有设施**: 部分边可能已存在（权重为 0）

**扩展**:
- **Steiner Tree**: 允许添加中间节点（NP-hard）
- **度约束 MST**: 限制节点最大度数
- **多目标 MST**: 同时优化成本、延迟等

class UnionFind:
    def find(self, x):
        if parent[x] != x:
            parent[x] = self.find(parent[x])  # 路径压缩
        return parent[x]
    
    def union(self, x, y):
        px, py = self.find(x), self.find(y)
        if px == py:
            return False
        # 按秩合并
        parent[py] = px

### 2.5 变邻域搜索 (Variable Neighborhood Search, VNS) - 详细实现

**核心思想**:
- 系统性地改变邻域结构，避免陷入局部最优
- 通过不同邻域的切换，实现全局搜索与局部搜索的平衡
- "如果当前邻域找不到更优解，就尝试更大的邻域"

**算法流程**:
```
1. 初始化当前解 x
2. 设置 k = 1 (从最小邻域开始)
3. while 未满足终止条件:
     a. 扰动 (Shaking): 在 x 的第 k 个邻域内随机生成 x'
     b. 局部搜索：在 x' 的邻域内搜索最优解 x''
     c. 判断：
        - 如果 f(x'') < f(x): 接受 x = x'', 重置 k = 1
        - 否则：k = k + 1 (尝试更大的邻域)
        - 如果 k > k_max: k = 1 (循环)
4. 返回最优解
```

**关键特点**:
- **扰动 (Shaking)**: 随机性帮助跳出局部最优
- **局部搜索**: 确定性搜索保证邻域内最优
- **邻域切换**: 系统性地探索不同邻域结构

---

#### 邻域结构设计 (TSP/路径问题)

**N1: 交换相邻 (Swap Adjacent)**
```
交换路径中相邻的两个节点
复杂度：O(n) 个邻居
```
示例：`[0,1,2,3,4]` → 交换 idx=2 → `[0,1,3,2,4]`

**N2: 交换任意 (Swap Any)**
```
交换路径中任意两个节点
复杂度：O(n²) 个邻居
```
示例：`[0,1,2,3,4]` → 交换 idx=1,3 → `[0,3,2,1,4]`

**N3: 路径逆转 (Reverse Segment)**
```
逆转路径中的一段
复杂度：O(n²) 个邻居
```
示例：`[0,1,2,3,4,5]` → 逆转 2-4 → `[0,1,4,3,2,5]`

**N4: 插入操作 (Insert)**
```
将一个节点移动到另一位置
复杂度：O(n²) 个邻居
```
示例：`[0,1,2,3,4]` → 插入 3→1 → `[0,3,1,2,4]`

**N5: 2-opt**
```
经典 TSP 邻域，逆转两段之间的路径
复杂度：O(n²) 个邻居
```
示例：`[0,1,2,3,4,5]` → 2-opt(2,4) → `[0,1,4,3,2,5]`

---

#### 参数调优

| 参数 | 推荐范围 | 说明 |
|------|----------|------|
| k_max | 3-7 | 邻域数量，太少易陷局部最优，太多计算慢 |
| max_iterations | 500-2000 | 最大迭代次数 |
| max_no_improve | 50-200 | 无改进最大迭代次数（提前终止） |
| 局部搜索深度 | 全搜索/部分搜索 | 全搜索质量好但慢 |

**调优策略**:
1. **从小邻域开始**: k=1 计算快，先尝试小改进
2. **渐进扩大**: 找不到改进时扩大邻域
3. **及时重置**: 找到改进后重置 k=1，充分利用小邻域

---

#### 优缺点分析

**优点**:
- ✅ 简单易实现，参数少
- ✅ 系统性地避免局部最优
- ✅ 适用于多种优化问题（TSP、VRP、调度等）
- ✅ 理论基础强（Mladenović & Hansen, 1997）
- ✅ 可扩展为 General VNS、Variable Neighborhood Descent

**缺点**:
- ❌ 邻域设计需要领域知识
- ❌ 局部搜索可能耗时（尤其是大邻域）
- ❌ 收敛速度可能慢于某些元启发式
- ❌ 参数 k_max 选择影响性能

---

#### 线缆布线应用

**场景**: 多设备顺序布线（类似 TSP）

**问题建模**:
- **解表示**: 设备访问顺序的排列
- **目标函数**: 总布线长度（或成本）
- **约束**: 所有设备必须访问一次

**优势**:
- 适合中等规模问题（20-100 节点）
- 能找到高质量近似解
- 可与其它算法混合（如 VNS+GA）

**实际应用建议**:
1. **初始解**: 使用最近邻启发式生成较好初始解
2. **邻域选择**: 从简单邻域开始（N1, N2），逐步增加
3. **混合策略**: VNS 后可接 2-opt 或 3-opt 进一步优化
4. **并行化**: 多个初始解并行运行 VNS

---

#### 代码实现要点

**VNS 主循环**:
```python
k = 1
while iteration < max_iter:
    # 1. 扰动
    new_sol = shake(current_sol, k)
    
    # 2. 局部搜索
    new_sol = local_search(new_sol, k)
    
    # 3. 判断是否接受
    if cost(new_sol) < best_cost:
        best_sol = new_sol
        k = 1  # 重置
    else:
        k += 1
        if k > k_max:
            k = 1
```

**局部搜索（最陡下降）**:
```python
def local_search(solution, k):
    best = solution
    best_cost = cost(solution)
    
    for neighbor in neighborhood(solution, k):
        if cost(neighbor) < best_cost:
            best = neighbor
            best_cost = cost(neighbor)
    
    return best
```

### 2.6 蚁群优化算法 (Ant Colony Optimization, ACO) - 详细实现

**核心思想**:
- 模拟蚂蚁觅食行为：蚂蚁在路径上释放信息素，后续蚂蚁倾向于选择信息素浓度高的路径
- **正反馈机制**：好路径 → 更多蚂蚁选择 → 更多信息素 → 更多蚂蚁
- **启发式引导**：蚂蚁也考虑距离等启发式因素（短路径优先）
- **信息素蒸发**：避免过早收敛到局部最优，保持探索能力

**灵感来源**：
> 蚂蚁个体能力有限，但群体通过信息素通信能找到从巢穴到食物的最短路径

**算法流程**:
```
1. 初始化信息素矩阵（所有边相同浓度）
2. while 未满足终止条件:
     a. 每只蚂蚁构建完整路径（概率选择下一节点）
     b. 计算每只蚂蚁的路径成本
     c. 更新信息素：
        - 蒸发：所有信息素 × (1 - rho)
        - 沉积：每只蚂蚁在其路径上留下 Q / cost
        - (可选) 精英蚂蚁额外强化最优路径
3. 返回历史最优解
```

---

#### 概率选择模型

**选择概率公式**:
```
P(i,j) = [τ(i,j)^α × η(i,j)^β] / Σ[k∈allowed] [τ(i,k)^α × η(i,k)^β]

其中:
- τ(i,j): 边 (i,j) 上的信息素浓度
- η(i,j): 启发式值，通常为 1/d(i,j)（距离的倒数）
- α: 信息素重要程度因子（α 越大越依赖信息素）
- β: 启发式因子重要程度（β 越大越贪心）
- allowed: 未访问节点集合
```

**参数影响**:
| 参数 | 值太小 | 值太大 | 推荐范围 |
|------|--------|--------|----------|
| α (信息素因子) | 退化为贪心算法 | 过早收敛到局部最优 | 0.5-2.0 |
| β (启发式因子) | 随机搜索 | 纯贪心，失去多样性 | 1.0-5.0 |
| ρ (蒸发率) | 收敛慢，信息素积累过多 | 失去记忆，退化为随机 | 0.1-0.5 |

---

#### 信息素更新策略

**1. 基本更新规则 (Ant System)**:
```
τ(i,j) = (1 - ρ) × τ(i,j) + Σ[k=1 to m] Δτ_k(i,j)

其中:
- ρ: 蒸发率 (0 < ρ ≤ 1)
- m: 蚂蚁数量
- Δτ_k(i,j) = Q / L_k  如果蚂蚁 k 经过边 (i,j)
             0        否则
- Q: 信息素强度系数
- L_k: 蚂蚁 k 的路径总长度
```

**2. 精英蚂蚁策略 (Elitist Strategy)**:
```
额外强化历史最优路径:
Δτ_best(i,j) = e × Q / L_best  如果 (i,j) 在最优路径上

其中 e 是精英蚂蚁数量（通常 5-10）
```

**3. 最大 - 最小蚂蚁系统 (MMAS)**:
```
限制信息素范围 [τ_min, τ_max]:
- 避免某些边信息素过高（垄断）
- 避免某些边信息素过低（被忽略）
- 定期重置信息素（停滞时）
```

---

#### 关键机制详解

**1. 正反馈 (Positive Feedback)**:
- 短路径 → 成本低 → 沉积信息素多 → 吸引更多蚂蚁
- 形成"富者愈富"效应，加速收敛

**2. 信息素蒸发 (Evaporation)**:
- 模拟自然界信息素挥发
- 作用：
  - 避免过早收敛（忘记坏路径）
  - 鼓励探索新路径
  - 适应动态变化的环境

**3. 启发式信息 (Heuristic Information)**:
- 提供问题特定的先验知识
- 对于 TSP/布线：η(i,j) = 1/d(i,j)
- 平衡"利用已知好路径"与"探索新路径"

**4. 精英策略 (Elitism)**:
- 额外强化历史最优解
- 加速收敛，但可能增加早熟风险
- 推荐精英蚂蚁数：5-10

---

#### 参数调优指南

**推荐初始配置**:
```python
config = ACOConfig(
    n_ants=30,          # 蚂蚁数量（≈ 节点数）

    alpha=1.0,          # 信息素重要性
    beta=2.0,           # 启发式重要性（通常 > alpha）
    rho=0.1,            # 蒸发率（10% 每代）
    Q=1.0,              # 信息素强度
    elite_ants=5,       # 精英蚂蚁数量

```

**调优策略**:
1. **先调 β**: 增大 β 使搜索更贪心，减小使搜索更随机
2. **再调 α**: 观察是否过早收敛，调整信息素影响力
3. **最后调 ρ**: 收敛太快→增大 ρ，收敛太慢→减小 ρ
4. **蚂蚁数量**: 通常设为节点数的 1-2 倍

**经验法则**:
- α=1, β=2-5 是常用组合
- ρ=0.1 适合大多数问题
- 精英蚂蚁数 = 总蚂蚁数的 10-20%

#### 线缆布线应用

**问题建模**:
- **节点**: 设备/接线盒位置
- **边**: 可能的布线路径
- **目标**: 访问所有设备一次，最小化总布线长度（TSP 变体）

**编码方式**:
```python
# 路径表示：节点访问顺序
solution = [0, 5, 3, 8, 1, 7, ...]  # 排列编码

# 信息素矩阵：n × n
pheromone[i, j] = 边 (i→j) 的信息素浓度

# 启发式矩阵：n × n
heuristic[i, j] = 1 / distance(i, j)
```

**实际应用技巧**:
1. **最近邻初始化**: 用贪心算法生成部分初始解，加速收敛
2. **局部搜索增强**: ACO 生成解后，用 2-opt 进一步优化
3. **自适应参数**: 根据收敛状态动态调整 α, β, ρ
4. **多起点策略**: 蚂蚁从不同起点出发，增加多样性

#### 代码实现要点

**ACO 主循环**:
```python
for iteration in range(n_iterations):
    # 1. 每只蚂蚁构建路径
    solutions = []
    for ant in range(n_ants):
        path = construct_solution(pheromone, heuristic, alpha, beta)
        solutions.append(path)
    
    # 2. 计算成本
    costs = [compute_cost(path, dist_matrix) for path in solutions]
    
    # 3. 更新历史最优
    best_idx = argmin(costs)
    if costs[best_idx] < best_cost:
        best_cost = costs[best_idx]
        best_solution = solutions[best_idx]
    
    # 4. 更新信息素
    pheromone *= (1 - rho)  # 蒸发
    for path, cost in zip(solutions, costs):
        delta = Q / cost
        for i in range(len(path)-1):
            pheromone[path[i], path[i+1]] += delta  # 沉积
    
    # 5. 精英蚂蚁额外强化
    if elite_ants > 0:
        elite_delta = elite_ants * Q / best_cost
        for i in range(len(best_solution)-1):
            pheromone[best_solution[i], best_solution[i+1]] += elite_delta
```

**路径构造（轮盘赌选择）**:
```python
def select_next(current, visited, pheromone, heuristic, alpha, beta):
    available = [i for i in range(n) if i not in visited]
    
    # 计算选择概率
    probs = []
    for node in available:
        tau = pheromone[current, node] ** alpha
        eta = heuristic[current, node] ** beta
        probs.append(tau * eta)
    
    # 归一化 + 轮盘赌
    probs = array(probs) / sum(probs)
    return choice(available, p=probs)
```

### 3.2 深度强化学习 (DQN/Q-Learning) - 详细实现

**核心思想**：通过与环境交互学习最优策略，无需先验知识

**强化学习基本框架**：
```
智能体 (Agent) ←→ 环境 (Environment)
    ↓                    ↓
  动作 (Action)       状态 (State)
    ↓                    ↓
                    奖励 (Reward)
```

**Q-Learning 算法**：

1. **Q 值定义**：
   - Q(s, a) = 在状态 s 执行动作 a 的期望累积奖励
   
2. **Bellman 方程**：
   ```
   Q(s,a) = Q(s,a) + α * [r + γ*max_a' Q(s',a') - Q(s,a)]
   ```
   - α (alpha): 学习率 (0.1-0.3)
   - γ (gamma): 折扣因子 (0.9-0.99)
   - r: 即时奖励
   - s': 下一状态

3. **ε-greedy 策略**：
   ```
   以概率 ε：随机探索 (exploration)
   以概率 1-ε：选择最优动作 (exploitation)
   ```
   - ε 随训练衰减 (1.0 → 0.01)

**DQN 改进**（完整版本）：
- 使用神经网络近似 Q 值函数 Q(s,a;θ)
- 经验回放 (Experience Replay)：打破样本相关性
- 目标网络 (Target Network)：稳定训练
- Dueling 架构：分离状态价值和动作优势

**线缆布线问题建模**：

| 组件 | 定义 | 说明 |
|------|------|------|
| 状态 (State) | 智能体位置 + 目标位置 + 障碍物地图 | 可离散化或连续表示 |
| 动作 (Action) | 上/下/左/右移动 | 离散动作空间 |
| 奖励 (Reward) | 到达终点 +10, 碰撞 -1, 每步 -0.1 | 鼓励快速到达 |
| 终止条件 | 到达终点/碰撞/超时 | 结束回合 |

**参数调优**：
- **学习率 α**: 0.1-0.3（太大不稳定，太小收敛慢）
- **折扣因子 γ**: 0.9-0.99（重视长期奖励）
- **探索率 ε**: 初始 1.0，衰减到 0.01
- **衰减系数**: 0.99-0.999 每回合

**优缺点**：
- ✅ 无需先验知识，从零学习
- ✅ 适应动态环境
- ✅ 可处理复杂状态空间（DQN）
- ❌ 训练时间长
- ❌ 样本效率低
- ❌ 超参数敏感

**与启发式算法对比**：
| 特性 | Q-Learning | GA/PSO | VNS/TS |
|------|------------|--------|--------|
| 学习方式 | 在线交互 | 种群进化 | 单解搜索 |
| 先验知识 | 不需要 | 不需要 | 不需要 |
| 训练时间 | 长 | 中等 | 短 |
| 适应性 | 强 | 中 | 弱 |
| 可解释性 | 中 | 中 | 高 |

**代码示例**: `examples/12_dqn_reinforcement_learning.py`

**实现提示**：
1. 状态离散化：连续空间 → 分箱 (binning)
2. Q 表初始化：零初始化或乐观初始化
3. 奖励塑形：稀疏奖励 → 稠密奖励（如距离奖励）
4. 经验回放：存储 (s,a,r,s',done) 元组

**测试结果**（10x10 网格，15 个障碍物）：
```

动作空间：4
最终平均奖励：-2.60
最佳奖励：10.00
测试平均步数：24.4
```

**收敛分析**：
- 前 50 回合：高探索率，随机搜索
- 50-150 回合：学习基本策略，奖励提升
- 150-300 回合：精细调优，稳定收敛

1. **Double DQN**: 解决 Q 值高估问题
2. **Dueling DQN**: 分离价值和优势
3. **Prioritized Replay**: 优先采样重要经验
4. **多智能体**: 多智能体协作布线
